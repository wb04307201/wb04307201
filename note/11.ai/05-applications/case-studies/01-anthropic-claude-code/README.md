# Anthropic 让 Claude Code 写 Claude Code：研究型公司如何让 Agent 跑满研发流水线

> **一句话总结**：Anthropic 工程师几乎不手写代码——Claude Code 创造者 Boris Cherny 自述一年多没写过一行，通过 `/loop`、MCP、Slack 跨 Agent 通信组成了一套"Agent 协作"的研发飞轮。

## 一、当 Agent 不再"辅助"工程师，而是"取代"工程师

业界对 AI 编程的想象，长期停留在"补全一行、生成一个函数"。但 Anthropic 内部正在发生的现实要激进得多。

Claude Code 创造者 Boris Cherny 在 2026 年 5 月 Sequoia AI Ascent 大会上承认：他"一年多没手写过一行代码了"。他用手机打开 App，同时挂 5 到 10 个会话，每个会话下又挂几百个子 Agent。最疯狂的一天，他通过 `/loop` 提交了 150 个 PR。

更值得玩味的是现场调查：在场几十位资深工程师里，"100% 手写的几乎没有，100% 用 Agent 写的也不多，绝大部分人在中间位置"。他的总结是——**"所以大概 50% 解决了。"**

这"50%"是当下 AI 编程行业最真实的写照：**Agent 已接管一半代码产出。** 在 Anthropic 这种"既是模型公司又是平台公司"的内部，那个 50% 正在逼近 100%。

---

## 二、不是"AI 帮你写代码"，而是"AI 帮你养一个 Agent 团队"

Claude Code 的颠覆性不在模型能力，而在**重新定义了"开发者"这个角色**。Boris 拿印刷术做类比：1400 年代欧洲识字率只有 10%，印刷术发明后 50 年内出版文献超过此前一千年。"软件也会经历同样的事情，速度比 50 年快得多。"

Anthropic 内部跑的是**多 Agent 并行协作的研发流水线**，核心组件有三层：

**会话级并行。** 手机里 5 到 10 个会话同时跑——一个"看护"PR 自动修 CI，一个保持 CI 健康，一个每 30 分钟从 X 抓用户反馈聚类整理。

**会话内子 Agent。** 每个主会话下又挂一层子 Agent。晚上用 Routine 让几千个在他关掉笔记本后继续跑。

**跨 Agent 通信。** 最反直觉的设计：**Anthropic 内部的 Agent 之间会"说话"**。当一个 Agent 卡住时，**通过 Slack 频道 @ 另一名工程师的 Agent**把问题丢过去。这种"Agent 间协作"拓扑，把组织里"问一句"的动作翻译成了 API 调用。

---

## 三、`/loop`：让 Agent 像后台守护进程一样工作

描述飞轮的机制只有一个：`/loop`。

原理：用 cron 调度定时任务。**Agent 从"按下回车后等结果"的工具，变成了"在后台永远跑着"的守护进程**。

Boris 举了一个例子：他让 Claude 拉一个数据查询，Claude 自己发现数据在随时间变化，主动说"我注意到数据在变，我会每 30 分钟给你发一份报告"。Boris 回了句"能发到 Slack 上吗？"，Claude 就自己调了 Slack 的 MCP 来推送。

**Agent 具备"自发觉察"能力**，识别"数据在变"并升级任务形态；**Agent 具备"自选工具"能力**，在没有用户提示的情况下自行调用 MCP。4.7 模型已经自发开始使用 `/loop`。

## 四、研究型组织的 review 机制：模型一样，组织不一样

Boris 对"内部比外面领先多少"的问题回答：

> **"在模型层面其实没有差距，我们用的模型和外部一样。差距在组织和流程上。"**

Anthropic 真正的领先优势，是把"模型 + 组织 + 流程"三件套重新设计了一遍。

**1. 严格的 dogfooding。** "Anthropic 做的是平台，特别在意 dogfooding，开发者用的和我们是同一套。"

**2. 跨职能写代码。** 工程经理、产品经理、设计师、数据科学家、用户研究员、连财务都在写代码——都在"指挥 Agent 写代码"。

**3. Agent 协作取代人协作。** 把"找人帮忙"动作抽象成 Agent 之间的 RPC 调用。

**4. Harness 持续缩小。** Boris 预测——**一年后 Claude Code 的产品外壳可能只需 100 行代码**。"模型强到一定程度，harness 几乎可以消失。" 这意味着 **"AI 写 AI"的飞轮也发生在产品迭代侧**。

---

## 五、对我们的启发：研究型公司如何让 Agent 输出不"失控"

读完整场对谈，最值得借鉴的是 **Anthropic 怎么把组织改造成 Agent 友好型的**。

**第一，把 Agent 当"同事"而不是"工具"。** Anthropic 内部不叫"用 Claude Code 写代码"，而是"我的 Claude 在干活"。

**第二，重视"跨 Agent 通信"。** Boris 强调的不是单个 Agent 多聪明，而是它们之间能否互通。**一个 Agent 卡住时，它能不能找到另一个 Agent 帮忙？**

**第三，用 `/loop` 思维重写后台任务。** 任何"定时执行"的场景——数据同步、日志巡检、CI 修复——都该问一句：**这件事能不能让 Agent 在后台永远盯着？**

**第四，dogfooding 必须做到极致。** 如果你自己都不敢天天用自己做的 Agent，客户凭什么用？

**第五，准备迎接"harness 缩小"的现实。** **产品形态会从"厚重的中间层"演化成"薄薄的连接器"。**

最后用 Boris 自己的话收尾：**"写代码的时代，或许真的，正在结束。但人人都可以'指挥 AI 写代码'的时代，才刚刚开始。"**
