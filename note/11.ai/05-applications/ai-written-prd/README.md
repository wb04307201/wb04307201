## 一、AI 写 PRD 的陷阱：越漂亮，越危险

当下，越来越多的公司开始用 AI 加速产出需求文档（PRD）。表面上看，AI 生成的 PRD 形式完整、措辞漂亮、逻辑通顺，几乎可以以假乱真。但问题恰恰出在这里——**它太像真的了**。

一份看似无懈可击的文档，很容易跳过质疑直接进入下游评审，甚至被开发团队当作"已经想清楚了"的信号开始执行。然而，漂亮的文字背后可能藏着：

- **空洞的假设**：没有数据支撑的市场判断
- **缺失的上下文**：对历史实验、跨团队依赖一无所知
- **未经验证的决策**：看似合理的产品方案，实则从未被挑战过

一旦这些问题被"漂亮的格式"掩盖，进入开发环节后造成的返工和资源浪费将是巨大的。

---

## 二、Uber 的选择：不让 AI 写 PRD，而是让 AI "审" PRD

面对这个问题，Uber 做了一个反直觉的决定：**不把 AI 做成"帮 PM 写 PRD"的工具，而是把它放在 PRD 进入正式评审之前，做第一轮预审。**

这个内部工具叫做 **PRD Evaluator**（需求文档评估器），它的角色定位非常明确：

> 不是替代资深人员的判断，而是帮助团队在进入高成本评审会议之前，带着更充分的上下文和更少的低级漏洞参与讨论。

换句话说，它坐在审批流程的上游，负责提升进入评审环节的文档质量。

## 三、PRD Evaluator 如何工作：四步从草稿到可执行评分卡

Uber 的 PRD Evaluator 并不是一个简单的"写作助手"，而是一套完整的评审系统。它的工作流程分为四个步骤：

### 第一步：围绕 PRD 构建更广的知识库

评估器以 PRD 为入口，利用 AI 搜索公司内部的相关文档和资料，自动构建一个上下文知识库。这些资料包括：

- 关联文档、相关演示文稿和会议纪要
- 历史实验数据
- 跨职能团队的输入材料
- Uber 内部预置的核心原则、指标定义和关键任务描述

这一步的关键在于：**不是只看 PRD 本身，而是把 PRD 放回它所处的业务语境中去理解。**

### 第二步：对 PRD 分类，校准评审深度

并非所有 PRD 都需要同等程度的审查。评估器会自动对每个提案进行分类，并相应调整评审力度：

| 变更类型 | 评审深度 |
|---------|---------|
| UX 一致性调整或可发现性优化 | 轻量级评审 |
| 增量式工作流变更或内部工具迁移 | 中等评审 |
| 全新功能开发 | 完整评审 |
| 涉及策略、定价或市场机制的变更 | 完整评审 + 专项审查 |

这种分级机制避免了"杀鸡用牛刀"，也防止了"大事化小"。

### 第三步：多维度评估上线准备度

评审围绕几个核心维度展开：

- **机会与假设**：问题是真实的吗？成功的定义是否足够清晰、可衡量？
- **产品范围**：提案是否易于理解、范围合理、可供决策？
- **用户体验与影响**：体验在不同用户群体、地域和边界场景下是否都能良好运作？
- **指标与数据严谨性**：PRD 是否定义了成功标准、防护指标和可信的验证方案？

### 第四步：生成面向行动的评分卡

评估器最终输出的不是一堆零散的批注，而是一份**结构化的评分卡**，包含：

- 上线准备度评级
- 各维度的逐项评估
- 一个清晰的"从这里开始"指引，指出最需要优先修复的问题
- 对每个缺口，说明缺失了什么，提供可直接替换的文本建议，并附上关联文档或历史实验中的证据
- 按优先级排序的行动项，分为"关键要求"和"优化建议"

## 四、早期验证：帮助 PM 发现盲区

从早期内部使用情况来看，PRD Evaluator 已经帮助数十位 PM 实现了以下价值：

- **尽早发现盲区**：在文档进入正式评审前就暴露问题
- **压力测试假设**：对那些缺乏数据支撑的"乐观估计"提出质疑
- **揭示相邻系统影响**：让 PM 看到自己负责范围之外的潜在连锁反应
- **发现已有范围内的体验改进空间**

更重要的是，当 PM 把这个工具融入日常的撰写和评审工作流后，进入评审会的文档质量明显提升，评审者也能把精力集中在更有价值的、高信号的讨论上。

## 五、启示：AI Agent 的正确打开方式

Uber 的这个案例给我们提供了一个非常有价值的思考角度：

**AI 在企业中的最佳落地点，往往不是"替代人做事"，而是"帮人把事做得更好"。**

当 AI 被用来写 PRD 时，它容易制造"虚假的完成感"；而当 AI 被用来审 PRD 时，它反而能成为提升决策质量的利器。这背后的逻辑是：

1. **AI 擅长处理信息密度**：它可以快速扫描大量关联文档，发现人类容易遗漏的上下文
2. **AI 擅长结构化评估**：它可以用统一的框架对文档进行多维度打分，避免评审的主观性
3. **AI 不替代决策**：最终的判断仍然由人来做，AI 只是确保进入决策环节的输入质量更高

在企业 AI 转型的过程中，与其追求"AI 自动完成某个任务"，不如思考"AI 可以在哪个环节帮助人做出更好的判断"。Uber 的 PRD Evaluator 正是后者的一个优秀范例。
