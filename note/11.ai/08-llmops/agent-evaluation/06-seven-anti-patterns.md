# 7 大反模式 · Agent 评测实战避坑

> **一句话**：Agent 评测 7 大常见坑都是"看起来对"——单一指标 / 一次性 / A/B 太短 / 黄金集不更新 / 阈值不合理 / 无漂移告警 / 测试集太干净。每个都是生产踩过的雷。

← [返回: Agent Evaluation 总目录](../README.md)

---

## 1. 反模式 1：只看单一指标

**错误**：只看任务完成率，忽略效率 / 成本 / 体验

**陷阱**：
- Agent 完成率 100% 但每任务 50 步 / $5
- 用户根本没耐心等完

**修复**：
- 必须 **6 维加权**（任务 40% + 步骤 20% + 工具 10% + 成本 10% + 满意 15% + 稳定 5%）
- 阈值 ≥ 0.80 才算通过

---

## 2. 反模式 2：上线前跑一次就发布

**错误**：评测一次高 → 发布 → 出事故

**陷阱**：
- 评测数据 ≠ 生产真实数据
- 任务成功 ≠ 用户满意

**修复**：
- 每次 PR 触发 CI 评估
- merge 后 A/B 灰度 5-50-100
- 在线持续监控

---

## 3. 反模式 3：A/B Test 时间不足

**错误**：A/B 跑 1-2 天就下结论

**陷阱**：
- 周末和工作日流量 / 用户行为差异大
- 1-2 天可能恰逢平静期

**修复**：
- ≥ **7 天**，覆盖工作日 + 周末 + 高峰期
- 显著性检验 p-value < 0.05

---

## 4. 反模式 4：黄金集不更新

**错误**：黄金集 1 年前标的，跟不上业务变化

**陷阱**：
- 新功能上线 → 黄金集没有对应任务
- 业务规则变 → 答案过时

**修复**：
- **每周**从用户反馈补充失败用例
- **每月** review 时效性
- **每季度**清理过期 + 补充新场景

---

## 5. 反模式 5：阈值不合理（100% 通过）

**错误**：要求 100% 完成率 = 永远发不了版

**陷阱**：
- 过于严格的阈值阻塞迭代
- 团队失去改进动力

**修复**：
- 阈值：**任务完成率 ≥ 0.85 + 综合 ≥ 0.80**
- 不同任务类型不同阈值（5% 失败对医疗不可接受，对娱乐合理）

---

## 6. 反模式 6：漂移无告警

**错误**：在线运行时无监控，事故发生 3 天后才发现

**陷阱**：
- 行业事件 / 新法规 → Agent 行为退化
- 没人告警 → 投诉累积

**修复**：
- **每日快测 50 条 + KS 检验** 漂移检测
- p < 0.05 → 自动告警
- 可选：自动回滚到上一个稳定版本

---

## 7. 反模式 7：评测集太干净（无对抗）

**错误**：黄金集全是"正常 query"，Agent 过拟合容易过测

**陷阱**：
- 生产遇到乱序字段 → 失败
- 用户拼写错误 → 失败
- 极端边缘 case → 失败

**修复**：
- **注入对抗样本**：
  - 字段顺序打乱（"我的姓名是 30 岁"）
  - 拼写错误（"麦子"代替"麦子"）
  - 模糊 query（"那个……能帮我订一下吗"）
  - 缺失字段（"我想订机票" → 缺日期 / 城市）

---

## 8. 7 反模式对比表

| 反模式 | 严重度 | 修复成本 |
|--------|-------|---------|
| 1 单一指标 | 高（效率 / 体验差）| 低（加权重） |
| 2 一次性评估 | 高（事故风险） | 中（CI 集成）|
| 3 A/B 时间不足 | 中（误判） | 低（延长） |
| 4 黄金集不更新 | 高（业务脱节） | 中（每周 review） |
| 5 阈值 100% | 中（阻塞发布） | 极低（调阈值）|
| 6 无漂移告警 | 高（事故漏报）| 中（监控集成）|
| 7 测试集太干净 | 中（上线后翻车）| 中（注入对抗）|

---

## 9. 一句话总结

> **7 反模式 = 单一指标 / 一次性 / A/B 太短 / 黄金集不更新 / 100% 阈值 / 无漂移告警 / 测试集太干净。每个都是生产踩过的雷，记住一句：评测不是发版前的仪式，是 24/7 的体系。**
