目前AI的应用场景正从单一问题的解决转向复杂问题的解决，`单智能体 vs 多智能体`就像`瑞士军刀 vs 工具箱`
- 单智能体 = 一把瑞士军刀，什么都能干但都不精
  - 打开所有工具时互相干扰，刀刃可能划伤自己
- 多智能体 = 一整套专业工具箱  
  - 螺丝刀专注拧螺丝、锤子专注敲钉子、尺子专注测量
  - 需要时取出对应工具，用完放回

# 单智能体的主要瓶颈：
1. 上下文膨胀问题
   - Prompt 超过一定长度（如 128K）后推理质量断崖式下降
   - 受上下文窗口限制，无法处理超长任务
2. 能力耦合问题
   - 单一 Agent 加载过多工具导致冲突和幻觉
   - 无法实现专业化分工
   - 容错脆弱问题
3. 单点失败导致整条链路中断
   - 缺乏降级和重试机制
   - 迭代成本高问题
   - 修改一个 Prompt 影响全局行为
   - 难以局部优化和热更新
4. 无法并行执行
   - 以单一进程顺序执行任务
   - 在重复上下文中浪费 Token
5. Token 效率低
   - 保持统一上下文和连续状态
   - 每一步都基于完整历史信息进行决策，造成资源浪费
# 多智能体系统的解决方案：
- 状态分片 + 按需路由：解决上下文膨胀
- 职责解耦 + 专业化工具集隔离：解决能力耦合
- 异步重试/降级策略/多体交叉验证：提升容错性
- 局部替换 + 拓扑可热更新 + 版本化编排：降低迭代成本
- 并行处理 + 专业化分工：突破单模型限制，提升效率
