## 📌 前言

Skills 已成为 Claude Code 中使用最广泛的扩展方式之一。它们灵活、易于创建，也方便分发。但这种灵活性也带来了一个问题：很难判断什么才是最佳实践。什么类型的 Skills 值得开发？写出一个优秀 Skill 的秘诀是什么？又应该在什么时候将它们分享给他人？

在 Anthropic 内部，我们已经在 Claude Code 中广泛使用 Skills，目前有数百个 Skills 在实际运行中。以下是我们在使用 Skills 来加速开发过程中总结的经验教训。

---

## 🔍 什么是 Skills？

> ⚠️ 常见误解：Skills「只是一些 Markdown 文件」

**事实**：Skills 最有趣的地方在于——它们并不仅仅是文本文件，而是一个**文件夹**，里面可以包含：
- 脚本（scripts）
- 资源文件（assets）
- 数据文件（data）
- 配置文件

这些内容都可以被 Claude Agent 发现、探索和操作。此外，Skills 还支持多种配置选项，包括注册动态钩子（hooks）等高级功能。

---

## 🗂️ Skills 的九大类型

通过整理内部数百个 Skills，我们发现它们大致可分为以下类别：

### 1️⃣ 库与 API 参考
解释如何正确使用库、CLI 或 SDK，包含参考代码片段和常见陷阱。
- 示例：`billing-lib`（内部账单库边缘情况）、`frontend-design`（设计系统规范）

### 2️⃣ 产品验证
描述如何测试或验证代码是否正常工作，常与 Playwright、tmux 等工具配合使用。
- 示例：`signup-flow-driver`（无头浏览器注册流程验证）、`checkout-verifier`（支付流程验证）

### 3️⃣ 数据抓取与分析
连接数据和监控系统，提供数据查询方式和标准工作流。
- 示例：`funnel-query`（用户转化漏斗查询）、`grafana`（仪表盘快速定位）

### 4️⃣ 业务流程与团队自动化
将重复工作流自动化为一个命令，适合标准化团队流程。
- 示例：`standup-post`（自动生成站立会议摘要）、`weekly-recap`（周报自动生成）

### 5️⃣ 代码脚手架与模板
生成特定功能的框架模板，适用于有自然语言要求的脚手架场景。
- 示例：`new--workflow`（新服务脚手架）、`new-migration`（数据库迁移模板）

### 6️⃣ 代码质量与审核
强制执行代码规范，辅助代码审查，可集成到 CI/CD 流程中。
- 示例：`adversarial-review`（对抗式代码评审）、`code-style`（风格检查）

### 7️⃣ CI/CD 与部署
帮助获取、推送和部署代码，支持自动化发布流程。
- 示例：`babysit-pr`（PR 自动监控与合并）、`deploy-`（渐进式发布与回滚）

### 8️⃣ 运行手册（Runbooks）
通过症状触发多工具调查，生成结构化故障报告。
- 示例：`-debugging`（服务故障排查指南）、`oncall-runner`（值班自动化响应）

### 9️⃣ 基础设施操作
执行常规维护操作，包含安全防护机制避免误操作。
- 示例：`-orphans`（孤立资源清理）、`cost-investigation`（成本异常分析）

## 🛠️ 制作 Skills 的 9 条最佳实践

### ✅ 不要陈述显而易见的内容
Claude 已经了解大量编码基础知识。你的 Skill 应专注于**让 Claude 脱离默认思维方式**的独特信息。

### ✅ 构建「常见陷阱」（Gotchas）部分
这是最有价值的内容。基于 Claude 实际使用中的失败点持续更新，确保技能的有效性。

### ✅ 善用文件系统和渐进式披露
Skills 是文件夹而非单文件。通过合理的文件结构组织上下文，让 Claude 在需要时读取相应内容。

### ✅ 避免「过度束缚」Claude
给予必要信息的同时，保留一定灵活性，让 Claude 能适应不同场景。

### ✅ 设计好 Skills 的初始化交互
如果 Skill 需要用户上下文（如 Slack 频道名），让 Claude 主动询问而非硬编码。

### ✅ 描述字段 = 触发场景说明
描述字段不是内容摘要，而是告诉模型**「什么情况下应该使用这个 Skill」**。

### ✅ 实现记忆与数据存储
通过日志文件、JSON 或 SQLite 存储数据，让 Skills 具备跨会话的记忆能力。

### ✅ 提供可执行脚本
将脚本和库交给 Claude，让它专注于组合决策而非重复造轮子。

### ✅ 按需钩子（On-Demand Hooks）
使用仅在 Skill 被调用时激活的钩子，适合有强烈意见但不希望全局生效的规则。

## 📤 Skills 的分发与协作

### 两种分享方式：
1. **代码库内部分享**：将 Skills 放入 `./.claude/skills` 目录并提交到版本库
2. **插件市场分发**：创建 Claude Code Plugin，用户可通过市场安装

### 市场管理建议：
- 无需集中审批，通过自然传播发现优质 Skills
- 先在沙箱文件夹分享，获得关注后再提交到正式市场
- ⚠️ 警惕冗余 Skills，发布前做好筛选和策划

### 组合 Skills：
虽然目前没有内建的依赖管理，但你可以直接通过名称引用其他已安装的 Skills，实现功能组合。

- 持续优化技能设计

## 🔑 核心总结

| 关键理念 | 实践建议 |
|---------|---------|
| Skills 是文件夹，不是单文件 | 用文件系统组织上下文，渐进式披露 |
| 描述字段决定触发时机 | 写清楚「何时用」而非「是什么」 |
| 陷阱部分最有价值 | 持续收集失败案例并更新 |
| 灵活胜于僵化 | 给 Claude 决策空间，避免过度约束 |
| 分享促进迭代 | 通过小范围测试→市场发布的流程筛选优质技能 |

> 💡 **一句话精髓**：80% 的时间花在规划 Skill 的设计和使用场景，20% 交给 Agent 执行。`plan.md` 是一切的锚点——当上下文丢失或会话中断时，它能帮助快速恢复。
