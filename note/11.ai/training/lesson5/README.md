## 课程导语

在前几课中，我们学习了 MCP（Model Context Protocol）——它解决了 AI Agent **"能做什么"** 的问题，让大模型能够连接外部 API、数据库和各种工具资源。然而，"能调用工具"只是第一步，真正的挑战在于：**如何将这些能力组织起来，高效、可靠地完成复杂任务？**

这就是 Skill 的价值所在。如果说 MCP 是 AI Agent 的"工具箱"，那么 Skill 就是"操作手册"——它封装了领域知识、工作流程和最佳实践，告诉 AI **"怎么做"** 才能把事情做对、做好。

| 维度 | MCP（工具层） | Skill（技能层） |
|:-----|:-------------|:---------------|
| 核心问题 | 能做什么？ | 怎么做？ |
| 本质 | 标准化的工具调用协议 | 可复用的能力封装单元 |
| 关注点 | 连接与调用 | 编排与质量 |

本课分三章递进展开：**第一章**建立 Skill 的概念与架构认知；**第二章**掌握五种核心设计模式，学会根据场景选择合适的模式；**第三章**走进 Claude Code 团队的真实工程实践，看 Skills 如何在生产环境中落地。学完后，你将具备设计和编写自己第一个 Skill 的能力。
