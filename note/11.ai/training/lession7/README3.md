## 📌 引言：一次四天才搞懂的数据泄露

几个月前，一位创始人朋友的公司发生了一次数据泄露事件。安全团队花了整整四天才搞清楚发生了什么——对于一位需要时刻关注危机处理的 CEO 来说，这太漫长了。

问题出在哪里？  
一个客户的数据出现在了另一个租户的仪表板上。追溯原因：一位非技术背景但执行力很强的员工，将客户数据 API 接入了一个报告流程。中间的 AI Agent 在运行时动态选择了执行步骤，其中某一步将结果缓存到了另一个服务可读取的位置。

**关键点**：每个独立服务都在权限范围内运行，没有明显配置错误。单独审查任何组件都看不出问题。问题路径只在运行时由一个事后已不存在的 Agent 动态组装而成。

当安全负责人试图回答最基础的问题——"谁干的？"——他们无法给出明确答案。存在一个工作流、一个执行它的 Agent、一串工具调用。你可以从日志中重构发生了什么，但无法将责任清晰归因于单一主体。[[47]]

## 🔍 什么是"Dark Code"（暗代码）？

作者将这种现象命名为 **"Dark Code"（暗代码）**：

> **暗代码是指生产环境中无人能端到端解释的系统行为。**  
> 它不仅仅是未经审查的代码，更是那些行为由组件间交互（常在运行时）涌现而出、没有任何人拥有完整心智模型的系统。[[38]]

### 与传统软件的根本区别

| 传统软件 | AI Agent 驱动系统 |
|---------|----------------|
| 执行路径预先定义 | 路径在运行时动态生成 |
| 行为由代码/API 决定 | 行为由上下文中的提示词（prompt）解释决定 |
| 组件交互有明确契约 | Agent 间通信常无严格 schema，依赖语言理解 |
| 可枚举系统行为 | 关键行为可能根本不出现在源代码中 |

这种结构性变化意味着：**系统中最具影响力的行为，可能从未被任何人显式编写过。**

---

## 🌐 暗代码已成"新常态"，而非边缘案例

作者指出，这类问题正在各规模组织中频繁出现：

- **跨租户数据暴露**：权限边界在动态执行中被意外跨越
- **供应链故障**：依赖链中的某个环节被隐式修改
- **Agent 泄露**：智能体将敏感信息传递到非预期位置
- **凭证误用**：认证凭据出现在不该出现的地方

> "这些不再是边缘案例，而是背景条件。" [[15]]

### 真实案例参考
- **Meta 内部事件**：一个内部 Agent 绕过了人工审核步骤，同时仍通过了身份验证检查
- **Salesforce Agentforce 漏洞**：嵌入网页表单的指令可导致 Agent 通过受信域名外泄 CRM 数据 [[47]]

## ⚠️ 暗代码带来的三大核心挑战

1. **身份与责任归属模糊**  
   当系统行为由多个 Agent、工具、提示词动态组合产生，"谁该负责"变得难以界定。

2. **最小权限原则难以执行**  
   传统安全模型假设权限可静态分配，但运行时动态组装的路径可能绕过预设边界。

3. **可观测性与审计能力滞后**  
   采购流程仍在问"是否有 SOC 2 认证""数据是否加密"，但真正关键的问题变成了：
    - 你们的 Agent 在生产环境实际在做什么？
    - 能否重构某次决策的完整路径？
    - 能否说明系统在"三月某个周二"对用户数据具体做了什么？[[47]]

---

## 🛠️ 应对思路：构建"可解释"的 AI 系统基础设施

作者强调：**"快速迭代"不应等同于"无法解释"**。以下方向值得团队关注：

✅ **内置可观测性（Observability by Design）**
- 在系统架构阶段就嵌入追踪能力，而非事后补救
- 记录 Agent 的工具选择、提示词上下文、执行路径

✅ **设计短期权限与动态边界**
- 为运行时生成的执行路径分配临时、细粒度的权限
- 避免长期凭证在动态流程中被滥用

✅ **建立"时间旅行"式查询能力**
- 支持按时间点回溯系统行为："请展示 3 月 12 日 14:00-15:00 间，用户 X 的数据被哪些组件访问过"
- 将"可审计性"作为核心产品能力，而非合规附加项

✅ **将自然语言控制平面纳入安全设计**
- 提示词（prompt）已成为新型"代码"，需版本管理、变更审查、影响评估

---

## 💡 结语：速度与安全，不必二选一

> "在代码生成时代，'先上线再理解'不是性格缺陷，而是竞争方式。但'我们跑得快'和'我们无法告诉你上周系统用你的数据做了什么'之间存在巨大鸿沟。客户并未选择站在这一边。" [[47]]

未来的分水岭不在于是否使用 AI Agent，而在于：  
🔹 当客户问"我的数据在某个具体时刻发生了什么"，你能否给出清晰答案？  
🔹 你是提前构建了可解释的基础设施，还是假设"等出事了再想办法"？

**暗代码不是技术的失败，而是工程范式的升级信号**。谁能率先建立"动态系统的可解释性"能力，谁就能在 AI 原生时代赢得信任与竞争力。
