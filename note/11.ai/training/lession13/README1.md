## 📌 核心摘要

本文分享了 Anthropic 团队在构建**长期运行的自主编码代理系统**方面的实践经验。核心创新在于借鉴 GAN（生成对抗网络）思想，设计了一套「生成器-评估器」多智能体架构，通过**结构化任务分解**、**上下文重置机制**和**可量化的评估标准**，显著提升了 Claude 在前端设计和全栈应用开发中的输出质量。

**关键成果**：
- ✅ 前端设计：通过 5-15 轮迭代，生成具有独特审美风格的网页，突破"模板化"瓶颈
- ✅ 全栈开发：从一句话需求出发，自主构建包含 16+ 功能的完整应用（如复古游戏编辑器、浏览器 DAW）
- ✅ 架构演进：从复杂的多智能体协作，逐步简化为更轻量的方案，适应模型能力的提升

## 🔍 一、为什么"简单方案"会失败？

在长期运行的代理任务中，两个关键问题限制了效果：

### 1️⃣ 上下文焦虑（Context Anxiety）
- 随着任务进行，上下文窗口逐渐填满，模型倾向于**提前结束任务**或**失去逻辑连贯性**
- 传统"压缩"（compaction）策略仅总结历史，无法提供"干净起点"
- **解决方案**：采用**上下文重置**（context reset）——清空对话历史，通过结构化工件（artifact）传递状态和下一步任务

### 2️⃣ 自我评估偏差（Self-Evaluation Bias）
- 模型倾向于**高估自己产出的质量**，尤其在主观任务（如设计）中
- 即使有客观验证标准，代理仍可能因"过度自信"忽略关键缺陷
- **解决方案**：将「执行」与「评估」分离，训练独立的评估器代理，通过外部反馈驱动迭代

> 💡 核心洞察：**分离关注点**（Separation of Concerns）是提升长期任务可靠性的关键杠杆。

## 🎨 二、前端设计：让"主观审美"可量化

### 评估框架：四大核心维度

| 维度       | 评估要点                                  | 权重策略     |
|----------|---------------------------------------|----------|
| **设计质量** | 色彩、排版、布局是否形成统一的视觉语言和情绪表达              | ⭐⭐⭐⭐ 高权重 |
| **原创性**  | 是否有定制化创意，而非模板/默认组件/"AI 套路"（如紫色渐变+白卡片） | ⭐⭐⭐⭐ 高权重 |
| **工艺水平** | 字体层级、间距一致性、对比度等技术执行细节                 | ⭐⭐ 基础要求  |
| **功能性**  | 用户能否理解界面、找到主操作、完成任务                   | ⭐⭐ 基础要求  |

### 迭代工作流

```
用户提示 → 生成器创建前端 → 评估器通过 Playwright 实操页面 → 
输出评分+详细反馈 → 生成器决定"优化"或"转向" → 下一轮迭代
```

**实践细节**：
- 每轮迭代 5-15 次，单次运行最长 4 小时
- 评估器通过 Playwright MCP **主动操作页面**（截图、点击、导航），而非静态评分
- 生成器根据反馈动态决策：分数趋势好则深化当前方向，否则大胆切换美学风格

### 典型案例：荷兰艺术博物馆网站
- 第 9 轮：传统深色主题落地页，视觉精致但预期之内
- **第 10 轮**：彻底重构为 3D 空间体验——CSS 透视棋盘格地板、自由悬挂的画作、基于"门"的房间导航
- ✨ 这种**创造性跃迁**在单次生成中几乎不可能出现，正是迭代反馈的价值体现

## ⚙️ 三、扩展到全栈开发：三智能体架构

### 架构设计

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Planner   │────▶│  Generator  │────▶│  Evaluator  │
│  (规划器)   │     │  (生成器)   │     │  (评估器)   │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       ▼                   ▼                   ▼
• 1-4句需求扩展     • 按特性分冲刺开发    • Playwright 实操测试
• 输出完整产品规格  • React+Vite+FastAPI  • 基于合同验收标准评分
• 嵌入 AI 功能建议  • 每冲刺后自检+交接   • 未达标则返回详细修复建议
```

### 关键机制：冲刺合同（Sprint Contract）

由于产品规格保持高层抽象，**生成器与评估器在编码前协商"完成定义"**：
1. 生成器提出：本冲刺实现哪些功能 + 如何验证成功
2. 评估器审核：验收标准是否可测试、是否对齐规格
3. 双方迭代直至达成一致，再开始编码

> 📁 通信方式：通过文件系统传递状态（A 写文件 → B 读取响应 → A 继续），避免上下文污染

### 实测对比：复古游戏编辑器

| 方案 | 运行时长 | 成本 | 核心结果 |
|------|----------|------|----------|
| **单代理** | 20 分钟 | $9 | 界面布局浪费空间、工作流僵硬、**游戏无法运行**（实体无交互） |
| **完整 Harness** | 6 小时 | $200 | 全屏适配、视觉统一、**游戏可玩**、含 AI 辅助生成关卡/精灵 |

**评估器发现的关键问题示例**：
```markdown
• 矩形填充工具：仅在拖拽起止点放置瓦片，未调用 fillRectangle 函数
• 删除实体生成点：条件判断逻辑错误，需改为 `selection || (selectedEntityId && activeLayer === 'entity')`
• API 路由顺序：FastAPI 将 `/reorder` 误解析为 frame_id 整数，返回 422 错误
```

## 🔄 四、架构迭代：从复杂到精简

### 简化原则
> "找到最简单的可行方案，仅在必要时增加复杂度"

### 关键优化步骤

#### 1️⃣ 移除"冲刺"分解结构
- **背景**：Claude Opus 4.6 在长任务规划、代码审查、长上下文检索方面显著提升
- **效果**：生成器可连续运行 2+ 小时无需任务切分，减少编排开销
- **评估器角色变化**：
    - 对模型能力边界内的任务：评估器成为冗余开销
    - 对接近能力边界的任务：评估器仍提供关键质量保障

#### 2️⃣ 强化 AI 功能集成
- 通过针对性提示工程，使生成器能构建**真正的工具调用型代理**
- 应用内嵌 Claude 接口，用户可通过自然语言生成关卡、精灵、行为逻辑

### 新架构实测：浏览器 DAW（数字音频工作站）

**提示词**：`Build a fully featured DAW in the browser using the Web Audio API.`

**运行数据**：
| 阶段 | 时长 | 成本 |
|------|------|------|
| Planner | 4.7 分钟 | $0.46 |
| Build + QA（3 轮迭代） | ~3.8 小时 | $124.24 |
| **总计** | **~4 小时** | **$124.70** |

**成果亮点**：
- ✅ 完整的编排视图、混音器、播放控制在浏览器中运行
- ✅ 通过自然语言提示，代理可自主设置节拍/调性、编写旋律、添加鼓点、调整混音
- ⚠️ 局限：音频录制仍为桩实现、部分交互（如剪辑拖拽）未完全实现、模型无法"听"音乐导致审美评估受限

## 💡 五、经验总结与未来展望

### 可复用的核心原则

1. **实验驱动**：始终在真实任务上测试模型表现，通过日志分析持续调优
2. **任务分解**：复杂任务拆分为子问题，分配专业化代理处理
3. **动态适配**：新模型发布后，重新评估架构组件的"承重性"，移除冗余、增强新能力
4. **评估器调优**：评估器不是开箱即用的，需通过"对比人类判断→修正提示"的循环迭代

### 关于模型演进与 Harness 的关系

> "随着模型能力提升，有趣的设计空间不会缩小，而是**发生迁移**。"

- 短期：某些问题会因模型原生能力增强而"自动解决"
- 长期：更强的模型为**更复杂的 Harness 设计**提供基础，解锁此前无法实现的任务

**工程师的核心价值**：持续探索模型能力与架构设计的**新颖组合点**，在"简化"与"增强"之间找到动态平衡。

```markdown
# RetroForge - 2D 复古游戏编辑器

## 概述
RetroForge 是一个基于网页的创意工作室，用于设计和构建 2D 复古风格视频游戏。
它将 8/16 位经典美学的怀旧魅力与现代直观的编辑工具相结合，
使从爱好者到独立开发者的任何人都能通过自然语言交互快速原型、视觉迭代并分享创作。

## 核心模块
1. 项目仪表盘：创建/管理/预览游戏项目
2. 瓦片关卡编辑器：拖拽式世界构建 + AI 辅助生成
3. 像素精灵编辑器：支持动画帧、调色板、缩放控制
4. 实体行为系统：可视化逻辑编排（事件→动作）
5. 即时测试模式：一键运行游戏，实时调试
6. AI 集成：通过 Claude 生成精灵、关卡、行为配置

## 技术栈
- 前端：React + Vite + Canvas API
- 后端：FastAPI + SQLite/PostgreSQL
- 游戏引擎：自定义轻量级 2D 运行时
- AI 接口：Claude Agent SDK + 工具调用
```
