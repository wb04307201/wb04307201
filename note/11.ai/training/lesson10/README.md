# 第 10 课：Agent 评估方法论

> 建立有效的 Agent 评估体系，避免"生产环境救火"。不知道 Agent 表现好不好？你需要一套评估方法论。

---

## 学习目标

学完本课后，你将能够：

- 理解为什么 Agent 评估比传统软件测试更复杂
- 掌握评估体系的基本结构（Task → Agent → Grader）
- 区分三类 Grader（确定性检查、模型评分、人工评估）的适用场景
- 使用 8 步框架评估 Agent 的 Skill 质量
- 理解 pass@k 和 pass^k 等评估指标
- 从零到一搭建团队的第一套 Agent 评估流程

## 核心概念速查

```text
评估体系基本结构：
  Task（任务定义）→ Agent（执行者）→ Output（输出）→ Grader（评判）→ Score（得分）

三类 Grader：
  ├── 确定性检查：正则、关键词、代码执行（快、准、但覆盖面窄）
  ├── 模型评分：用 LLM 评判 LLM 输出（灵活、但有偏差）
  └── 人工评估：最准确、但最慢最贵
```
