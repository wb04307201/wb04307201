# 📝 Teddy Riker《Designing for Agents》推文内容整理

> **作者**：Teddy Riker（Ramp 产品负责人）  
> **来源**：X (Twitter) - https://x.com/teddy_riker/status/2047312986696454584  
> **主题**：在 AI Agent 时代，如何为智能体设计软件产品

---

## 🔑 核心观点

> "如果你常在 X 上关注 AI 动态，可能见过'UI 已死'的观点：除非产品能通过 MCP、API 或 CLI 被智能体使用，否则将无法生存。"

**关键洞察**：
- Ramp 过去三个月 MCP 周活跃用户增长 **10 倍**
- Salesforce 宣布"Headless 360"转型：将所有能力暴露为 API/MCP/CLI，让 AI Agent 无需浏览器即可操作整个系统
- **80/20 法则已翻转**：未来 80% 的软件交互将通过 Agent 完成，而非人类直接操作

---

## 🔄 交互模式的三层演化

| 阶段       | 交互流程                       | 特点                    |
|----------|----------------------------|-----------------------|
| 传统模式     | `用户 → 界面 → 数据库`            | 人类直接操作图形界面            |
| Agent 介入 | `用户 → 用户的智能体 → 数据库`        | 智能体代表用户执行任务           |
| 双智能体协作   | `用户 → 用户智能体 → 软件智能体 → 数据库` | 两个 LLM 协同，软件智能体处理业务逻辑 |

## 🎯 为 Agent 设计的三大原则

### 1️⃣ 教会智能体"什么叫成功"

**案例对比**：
- ✅ **Notion MCP**：工具描述中明确要求调用方先获取 `notion://docs/enhanced-markdown-spec` 规范，确保格式正确
- ❌ **Slack MCP**：未提供格式规范，导致 Agent 输出格式混乱，用户需手动修正

**核心建议**：
> 不要假设调用方知道一切。把规范、约束、最佳实践主动"喂"给智能体，而不是让它去文档里找。

---

### 2️⃣ 构建有效的反馈循环

Ramp 的 MCP 可观测性实践：

| 机制                    | 作用                      | 价值              |
|-----------------------|-------------------------|-----------------|
| **强制 `rationale` 参数** | 每次工具调用需说明"为什么"          | 重建用户意图，发现真实需求模式 |
| **独立 feedback 工具**    | Agent 遇阻时主动报告"想做什么/卡在哪" | 获取精准的产品改进信号     |
| **工具特定 seed 参数**      | 捕获 Agent 无法推断的上下文       | 为后续分析提供结构化数据    |

💡 **反直觉发现**：Agent 的反馈比人类用户更具体、更一致。例如："报告拉取了3天前不属于本次事件的工单"，可直接转化为产品迭代需求。

---

### 3️⃣ 留意"上下文鸿沟"（Context Gap）

**报销场景示例**：

```
👤 Diego 的 AI 助理掌握：
• 日历：会议时间、参与人
• 邮件：航班/酒店确认函
• Slack：晚餐与客户团队的关联
• 收据：从附件/相册提取

💼 报销系统 Agent 掌握：
• 交易原始数据（商家、时间）
• 公司报销政策
• 会计科目（GL codes）
• 历史编码模式
```

**传统 API 做法**：  
❌ "请选择 150 个会计科目中的一个" → 把问题抛回给用户

**Agent 友好设计**：  
✅ "这是一次客户用餐、团队聚餐还是个人差旅？" → 从上下文自动推断，用户无需懂会计科目

> 每一方贡献自己擅长的上下文，共同交付对用户和财务团队都更好的结果。

---

## ⚠️ 警示：避免"打勾式"开发

> "大多数公司会发布一个 MCP，打个勾，然后继续前进。使用量增长几个季度后就会停滞。"

**成功关键**：
- 用设计人类界面的同等心思，去设计 Agent 界面
- 关注细节：规范传递、反馈机制、上下文对齐
- 长期来看，客户会流向"把 Agent 体验做到极致"的产品

---

## 📌 一句话总结

> **"你的智能体调用者需要什么才能完成它的工作？你给了他们吗？"**  
> 在 Agent 时代，产品设计的核心从"让人会用"转向"让智能体能成功"。
