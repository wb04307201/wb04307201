上周我们说到了Agent Harness概念，目的是建设可长期运行的可靠系统。

比提到了一个方法论，本周我们一起来看一下什么是**控制论（Cybernetics）**

请先阅读
[工程技术：在智能体优先的世界中利用 Codex](README1.md)
阅读之后我们再回想一下实践建议：
1. **从小处着手**：先让智能体处理明确、边界清晰的小任务
2. **投资情境管理**：构建结构化、可导航的知识库，而非堆砌指令
3. **为智能体设计**：优化代码、日志、文档的"机器可读性"
4. **用约束换自由**：在关键边界强制执行规则，在实现细节允许自主
5. **建立自动治理**：将"品味"和"原则"编码为可执行的检查器
6. **接受迭代文化**：高吞吐系统中，快速纠错优于完美主义等待

这和Agent Harness的Harness的核心目的一样，我们要去控制，去驾驭

因此  
当我们不再亲手"写代码"，而是设计让智能体写代码的系统时，软件工程的本质正在回归一个古老而深刻的范式：**控制论（Cybernetics）**。  
[Harness Engineering Is Cybernetics](README2.md)  
到这里我们的当前使用AI的方法论也有定位，即**控制论**
