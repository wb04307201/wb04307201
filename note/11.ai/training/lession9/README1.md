## 🔍 核心理念：从"临时检索"到"知识积累"

当前大多数人使用 LLM 处理文档的方式是 **RAG（检索增强生成）**：上传文件 → 提问时检索相关片段 → 生成答案。这种方式的问题是：

> ❌ 每次提问，LLM 都要从零开始重新"发现"知识  
> ❌ 没有知识积累，无法形成系统性理解  
> ❌ 复杂问题需要综合多份文档时，效果大打折扣

**LLM Wiki 的核心创新**在于：

> ✅ LLM 不是临时检索，而是**增量构建并维护一个持久的 Wiki**  
> ✅ 这是一个结构化的、相互链接的 Markdown 文件集合，位于你和原始资料之间  
> ✅ 新资料加入时，LLM 会阅读、提取关键信息，并**整合到现有 Wiki 中**  
> ✅ 知识被"编译"一次后持续更新，而非每次查询都重新推导

**关键区别**：Wiki 是一个**持久化、可复利的知识资产**。交叉引用已建立，矛盾已标记，综合观点已融合。每添加一个来源、每问一个问题，Wiki 就变得更丰富。

> 🎯 你几乎不用亲手写 Wiki —— LLM 负责所有撰写和维护工作  
> 🎯 你负责：筛选资料、探索方向、提出好问题  
> 🎯 LLM 负责：总结、交叉引用、归档、维护一致性

## 🏗️ 三层架构设计

```
┌─────────────────────────┐
│   1️⃣ 原始资料层 (Raw)    │
│   • 你策划的文档集合      │
│   • 文章/论文/图片/数据   │
│   • 不可变，只读          │
└────────┬────────────────┘
         │ 读取
         ▼
┌─────────────────────────┐
│   2️⃣ Wiki 层 (核心)      │
│   • LLM 生成的 Markdown  │
│   • 摘要/实体页/概念页    │
│   • 交叉引用/对比/综合    │
│   • LLM 完全负责维护      │
└────────┬────────────────┘
         │ 遵循
         ▼
┌─────────────────────────┐
│   3️⃣ 模式层 (Schema)     │
│   • CLAUDE.md / AGENTS.md│
│   • 定义 Wiki 结构规范    │
│   • 指导 LLM 的工作流程   │
│   • 你与 LLM 共同演进     │
└─────────────────────────┘
```

## ⚙️ 三大核心操作

### 1️⃣ 摄入（Ingest）：添加新知识
```
你：放入新资料 → 告诉 LLM 处理
↓
LLM：
  • 阅读资料，与你讨论要点
  • 撰写摘要页并加入 Wiki
  • 更新索引和相关内容页
  • 在日志中记录变更
↓
结果：单个资料可能更新 10-15 个 Wiki 页面
```

> 💡 建议：初期逐个摄入并参与审核，熟练后可批量处理

### 2️⃣ 查询（Query）：获取答案
```
你：提出问题
↓
LLM：
  • 搜索相关 Wiki 页面
  • 综合信息生成带引用的答案
  • 答案形式：文本/表格/幻灯片/图表
↓
关键：优质答案可反向存入 Wiki，成为新知识资产
```

### 3️⃣ 检查（Lint）：维护健康
定期让 LLM 执行"健康检查"：
- 🔍 发现页面间的矛盾
- 🔄 标记被新资料推翻的旧观点
- 🔗 识别孤立页面或缺失的交叉引用
- ❓ 建议可深入探索的新问题
- 🌐 推荐可补充的外部资料

## 🗂️ 索引与日志：导航系统

### 📑 index.md（内容导向）
- Wiki 内容的目录清单
- 每页包含：链接 + 一句话摘要 + 元数据（日期/来源数等）
- 按类别组织：实体、概念、来源等
- LLM 每次摄入后自动更新
- 查询时先读索引定位，再深入阅读

### 📋 log.md（时间导向）
- 追加式操作日志：摄入/查询/检查记录
- 统一前缀格式便于解析：`## [2026-04-02] ingest | 文章标题`
- 提供 Wiki 演进的时间线
- 帮助 LLM 理解近期工作内容

## 💡 为什么这个模式有效？

### 人类放弃 Wiki 的根本原因：
> 维护知识库最累的不是阅读和思考，而是**琐碎的文书工作**：更新引用、保持摘要最新、标记矛盾、维护一致性……随着页面增多，维护成本呈指数级增长。

### LLM 的优势：
> ✅ 不会厌倦，不会忘记更新交叉引用  
> ✅ 一次操作可修改 15 个文件  
> ✅ 维护成本趋近于零，Wiki 能持续保持健康

### 分工明确：
```
👤 人类：策划资料 → 引导分析 → 提出好问题 → 思考意义
🤖 LLM： 总结 → 引用 → 归档 → 维护一致性 → 执行琐事
```

## ⚠️ 注意事项与实践建议

1. **本文档是抽象设计，非具体实现**
    - 目录结构、命名规范、页面格式、工具链都需根据你的领域定制

2. **模块化选择，按需采用**
    - 纯文本资料？无需图片处理
    - Wiki 规模小？index.md 足够，无需搜索引擎
    - 不需要幻灯片？忽略 Marp 即可

3. **与你的 LLM 协作迭代**
    - 将此模式分享给你的 Agent，共同定制适合你的版本
    - 文档的使命是传达"模式"，具体实现由你和 LLM 共同完成

4. **警惕幻觉污染**（社区反馈）
    - LLM 可能将错误信息写入 Wiki 并传播
    - 建议：关键事实保留来源引用，定期执行"检查"操作
