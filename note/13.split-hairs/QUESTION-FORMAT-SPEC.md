# 「咬文嚼字」面试题格式规范 v1.0

> 本规范定义 `note/13.split-hairs/**/*.md` 下所有 `.md` 面试题文件的统一写作格式。
> 适用范围：121 篇面试题（含 11.ai 下 14 篇、06.spring 下 14 篇等）。
> 适用对象：新写面试题、review 旧面试题、做 cheatsheet 自动化。

← [返回咬文嚼字 README](./README.md)

---

## 0. 为什么需要规范

2026 系列扩到 121 篇 + 2026 路径整理后，写作格式逐渐分散：

- 32 篇原来没有场景开头的"概念速记式" / "TL;DR 速记式"
- 不同子模块（11.ai / 09.front-end / 04.system-design）用了不同模板
- split-hairs README 自标的 6 节模板（核心原理 / 代码示例 / 陷阱 / 最佳实践 / 面试话术 / 交叉引用）覆盖率约 70%

规范不是束缚创作，而是：
1. 让"高频面试题列表"自动生成（cheatsheet）
2. 让读者第一眼"掉进问题里"（场景开头强制）
3. 让 cross-ref 工作（与主模块 [18 个目录] 的引用网络保持一致）

---

## 1. 文件结构（强制骨架）

按以下顺序排版，每节之间用 `---` 空行隔开：

| 区段 | 强制 | 用途 |
|------|-----|------|
| ① H1 标题 | **强制** | `# 标题 —— 一句话明确问题` |
| ② 一句话定位 blockquote | 推荐 | L3 `> 一句话定位：xxx —— 全篇 30 字点题`（11.ai 系列惯例） |
| ③ 系列定位块 | **强制** | `> 经典 X 面试题（Y、Z 高频）。考察的不是"…"，而是 **...** + **...** + **...**。` |
| ④（可选）上下文缩进 | 可选 | `⭐⭐⭐⭐ 深度级别`、`难度：⭐⭐⭐⭐`、`📚 前置知识：` |
| ⑤ 引子 | **✅ 强制** | 一个具体场景开场，详见 §2 |
| ⑥ 章节 1...N | 强制 | `## 一、核心原理`（默认），可省略变体见 §3 |
| ⑦ 面试话术 | 推荐 | `## X、面试话术（90 秒版本）` —— 一段可直接对答的术语 |
| ⑧ 交叉引用 / 相关章节 | 强制 | `## X、相关章节` —— 同栏目 + 主模块双向链接 |
| ⑨ 文末标注 | 可选 | `> 📅 YYYY-MM-DD · 咬文嚼字 · X主题 · ⭐⭐⭐⭐` |

### 完整骨架示例（基于已落地的 11.ai 系列）

```markdown
# 标题深度剖析：副标题

> 一句话定位：xxx 直接决定 X、Y、Z —— 是全栈工程师的底层基建。完整概念见 [主模块 X](../../../11.ai/02-technology-stack/x/README.md)。

> **系列定位**：经典 Java/AI/系统面试题（X、Y、Z 高频）。考察的不是"...", 而是 **A 层面 1** + **B 层面 2** + **C 层面 3**。

### 关键约束

- **H1 标题**：必须明确问题（"为什么慢"、"3 大坑"、"X 是什么"），不能模糊
- **引子强制**：缺失引子 = 违反规范 = 不可 review 通过
- **章节最少 5 节**：核心原理 / 代码示例 / 陷阱 / 最佳实践 / 面试话术 中至少 4 节存在
- **结尾交叉引用**：必须指向主模块对应位置（**不允许** 死链接 / `./` 裸路径）

## 2. 引子的 4 种合法场景类型

任何 `## 引子：xxx` 必须从下面 4 种选 1 种来构建（不可直接抛定义）：

### 2.1 生产 Bug（最推荐 50%+ 的题）

```text
🚨 凌晨 3 点告警：生产服务器 CPU 使用率 98%
```

**结构**：告警时间 + 现象 + 损失 + **疑问**。

### 2.2 反直觉代码（最推荐基础题）

```js
Integer a = 100; Integer b = 100; System.out.println(a == b);
```

跑出 true。但 `Integer a = 1000; Integer b = 1000` 却跑出 false。

**结构**：代码 + 反直觉现象 + **疑问**。

### 2.3 架构决策困境

```text
你有两台服务器做数据同步，网络突然断了——
```

CP（拒绝写入）还是 AP（两边都接受）？

**结构**：场景 + 后果对立 + **选择困境**。

### 2.4 性能对比（front-end / perf 题首选）

```text
第一次访问：2 秒
普通刷新：0.5 秒
再次访问：0.1 秒
```

**结构**：数字对比 + 提问"为什么差 20 倍"。

### 2.5 反例禁止

- ❌ "本文介绍 X 的原理..."（无场景）
- ❌ 直接列公式 / 列要点 / TL;DR 表格（缺铺垫）
- ❌ 只引用主模块一句话定位就完事（11.ai 系列旧的速记式）

## 3. 章节命名规范

### 3.1 默认 6 节模板（推荐基线）

```markdown
## 一、核心原理（Why it works）
## 二、代码示例（Show me the code）
## 三、常见陷阱（What goes wrong）
## 四、最佳实践（Do this instead）
## 五、面试话术（How to answer in 30s）
## 六、交叉引用（Link back to main module）
```

### 3.2 已有的合法变体（保留）

各子模块已经有 4 种合法变体，新写时**按章节适配**：

| 子目录 | 章节模式 | 备注 |
|--------|---------|------|
| 11.ai 系列 | `## 一、核心原理` / `## 二、面试陷阱` / `## 三、最佳实践` / `## 四、面试话术` | 11.ai 系列的实际模板（不是默认 6 节）|
| 09.front-end `xxx/*README.md` | `## 一、核心结论（TL;DR）` + `## 二、详解` + `## 三、避坑指南` | "TL;DR 速记式"用于审题型短题 |
| 03.database SQL 调优类 | 同 09.front-end 的 TL;DR 模式 | 沿用既有规范 |
| 番外/管理类（已迁出） | 不再使用 | 见 README §4 保留边界 |

> 所有变体**必须保留 `## 引子：xxx`**，且**至少含 `核心原理 + 常见陷阱 + 面试话术` 3 节**。

### 3.3 数字编号强制

**所有章节标题用中文数字**：`一、二、三、四、五、六`
**禁止** "`第 1 章`" "`Chapter 1`" "`1.`" "`一、` 单独编号" 混用。

---

## 4. 引用 / 衔接规范

- **同栏目链接**：`../xxx/README.md`（相对路径）
- **主模块链接**：`../../../xx/yy/README.md`（向上一两级，相对路径）
- **跨主模块链接**：`../../11.ai/...` 等
- **禁止**：绝对路径、纯文本 `02-xxx`、死链接、URL（GitHub blob）

---

## 5. frontmatter（可选推荐）

```markdown
<!--
question:
  id: hashmap-resizing
  topic: 01.java
  difficulty: ⭐⭐⭐⭐
  frequency: 高频
  scenario_type: 性能对比
  tags: [HashMap, 扩容, Java 8, 红黑树]
-->
```

不强制，但加上后可解析出"高频题列表"，与 [cheatsheet.md] 等自动联动。

## 6. 反例清单

| 反例 | 出处 | 修正 |
|------|------|------|
| 没有 `## 引子：xxx` 直接进入核心原理 | 32 篇（已批量修复于批 3 / 4 / 5） | 在标题块之后、`## 一、` 之前插入 |

| 主模块链接写在引子里 | 散见 | 主模块链接统一放在 §⑥ |
| 内容是"决策实战 / 选型指南"而非面试陷阱 | 4 篇决策类（已迁出） | 见 README §4 保留边界 |

## 7. 与 [`12.story`](../12.story/STORY-FORMAT-SPEC.md) 的对比

| 维度 | story（叙事） | split-hairs（刺刀） |
|------|--------------|-------------------|
| **目标读者** | 系统学习者 | 求职者 / 面试准备 |
| **节奏** | 章节式叙事（第一章/第二章） | 一题一文（引子 + TL;DR + 陷阱） |
| **首段** | 一句话副标题 | 一句话副标题 + 一个具体场景 |
| **深度** | 100-300 行 / 章节 | 50-150 行 / 主题 |
| **典型交叉引用** | `[主模块 ↔ 故事]` | `[split-hairs ↔ 主模块]` |
