# LLM 幻觉问题深度剖析

## 引子：AI 一本正经地胡说八道

```
用户：请介绍一下《红楼梦》的作者曹雪芹的生平。

LLM：
"曹雪芹（1715年-1775年），字梦阮，号雪芹。
他于1740年完成了《红楼梦》的创作，
该书共120回，后40回由高鹗续写..."

问题：曹雪芹生于1715年？1740年完成？后40回高鹗续写？
这些"事实"都是 LLM 编造的！
```

LLM 最让人头疼的问题：**幻觉（Hallucination）**——生成的内容语法流畅、看起来很专业，但事实是错的。

更可怕的是，它编造时**非常自信**，人类很难第一时间辨别。

---

## 一、核心原理

**什么是幻觉（Hallucination）**

幻觉是大语言模型（LLM）特有的现象，指模型输出的内容在语法上流畅、语义上连贯，但与客观事实不符、与源文档矛盾，或完全是"凭空捏造"的信息。其危害在于：**错误的信息往往以高度自信的语气呈现**，人类难以第一时间辨别真伪。

**幻觉的分类**

从验证角度，幻觉可分为两类：

| 类型 | 定义 | 示例 |
|------|------|------|
| **Intrinsic Hallucination**（内在幻觉） | 模型输出与给定的源文档/上下文不一致，存在逻辑矛盾或事实冲突 | RAG 场景中，检索到的文档说"A 公司成立于 2010 年"，但模型回答"2015 年" |
| **Extrinsic Hallucination**（外在幻觉） | 模型输出无法被源文档验证，属于"无中生有"的捏造 | 用户问"某小众电影的票房"，模型编造一个具体数字，而该数据在任何可靠来源中都不存在 |

从内容性质角度，还可分为：

- **事实性幻觉（Factual Hallucination）**：生成的内容与客观世界的事实不符，如错误的人物生平、虚构的历史事件。
- **忠实性幻觉（Faithfulness Hallucination）**：在 RAG 等需要依据上下文的场景中，模型未能忠实于提供的参考材料，添加了原文没有的信息或曲解了原意。

理解这两类幻觉的区别至关重要：**Intrinsic 幻觉可以通过对比源文档自动检测；Extrinsic 幻觉则需要外部知识库或人工核查。**

## 二、产生原因

幻觉的产生并非单一因素所致，而是训练数据、模型架构、解码策略三者共同作用的结果。

### 1. 训练数据偏差

LLM 的训练语料来自互联网爬取的海量文本，这些数据本身存在以下问题：

- **噪声与错误**：网页中包含大量未经验证的观点、过时的信息、甚至故意传播的谣言。模型在预训练阶段将这些错误内化为"知识"。
- **分布不均**：热门话题（如明星八卦）的数据远多于冷门领域（如小语种文献），导致模型对长尾知识的掌握薄弱，容易"靠猜"。
- **自相矛盾**：不同来源对同一事实的描述可能冲突，模型学到的是概率分布而非确定性真理。

### 2. 自回归生成的概率本质

LLM 本质上是下一个 token 的概率预测器（next-token predictor）。它在每一步选择"最可能的下一个词"，而非"最正确的下一个词"。这种机制带来两个根本问题：

- **局部最优陷阱**：模型追求的是序列级别的似然最大化，单个 token 的选择可能在局部合理，但累积起来导致全局性的事实偏离。
- **暴露偏差（Exposure Bias）**：训练时模型看到的是真实的前文，推理时它看到的是自己生成的前文。一旦早期出现微小错误，后续会沿着错误路径不断放大（error propagation）。

### 3. 知识截止日期与新事实

模型的参数中固化的是训练截止时刻的世界知识。对于截止之后发生的事件（如新上任的政治人物、最新发布的财报），模型要么完全不知道，要么用旧知识"脑补"出新答案。

### 4. 模型的"过度自信"

LLM 没有内置的"不确定性表达"机制。即使模型对某个事实的把握只有 30%，它仍会以与 99% 把握相同的语气输出。这是因为：

- **Softmax 的温度效应**：即使 logits 差异很小，经过 softmax 后也会产生一个相对最高的概率，模型必须选一个。
- **RLHF 的副作用**：人类偏好奖励的是"流畅且确定的回答"，而非"谨慎且承认无知的回答"，这进一步放大了模型的自信倾向。

## 三、检测与评估

### 1. 基于参考文档的验证（NLI / Entailment）

将模型输出与检索到的参考文档送入自然语言推理（NLI）模型，判断两者之间的关系：

- **Entailment（蕴含）**：输出可被文档支持 → 可信
- **Contradiction（矛盾）**：输出与文档冲突 → Intrinsic 幻觉
- **Neutral（中立）**：文档未提及 → 可能是 Extrinsic 幻觉

常用工具：SNLI/MNLI 数据集微调的模型、FactCC、QAFactEval。

### 2. 基于知识图谱的验证

将模型输出中的实体和关系抽取出来，与结构化知识库（如 Wikidata、DBpedia）进行比对。若三元组 `( subject, relation, object )` 在知识库中不存在或冲突，则标记为潜在幻觉。

优点是可解释性强，缺点是覆盖率受限于知识库的完备性。

### 3. 多模型交叉验证（Consistency Check）

对同一问题用多个独立模型（或同一模型多次采样）生成答案，比较结果的一致性。一致性越低，幻觉风险越高。这种方法尤其适用于没有参考文档的开放域问答。

### 4. 标准化评估基准

| 基准 | 说明 |
|------|------|
| **TruthfulQA** | 包含 817 个易诱导人类犯错的问题，评估模型在"反常识陷阱"上的表现 |
| **RAGAS Faithfulness Score** | 针对 RAG 场景，衡量生成答案中每个陈述是否都能被检索到的上下文支持 |
| **HaluEval** | 覆盖多选问答、开放问答、摘要等任务的幻觉检测基准 |
| **Human Evaluation** | 人工逐句标注事实正确性，仍是黄金标准 |

## 四、缓解方案

### 1. RAG 兜底：让模型"有据可依"

检索增强生成（RAG）是最主流的抗幻觉手段。其核心思想是：**不依赖模型参数中固化的知识，而是从外部检索实时、可信的文档作为生成依据**。

关键实践：
- 检索后对文档做重排序（Re-ranking），确保最相关的片段进入上下文窗口。
- 在 Prompt 中明确要求"仅根据以下材料回答，若材料不足则说明无法回答"。
- 生成后做引用检查（Citation Verification），确保每个陈述都有对应的出处标记。

### 2. Self-Consistency：多次采样投票

对同一问题生成 N 条独立答案（通过提高 temperature 增加多样性），然后投票选出最一致的结果。直觉是：**幻觉往往是随机的，而事实具有稳定性**。

适用场景：数学推理、代码生成等有明确"正确答案"的任务。缺点是无法处理开放域问题（多个答案可能都合理）。

### 3. 降低温度 + 置信度过滤

- **Temperature 降至 0~0.3**：减少采样随机性，使模型更倾向于选择高概率 token。
- **Top-p / Top-k 截断**：限制候选词范围，避免低概率的"离群"选择。
- **Logits 置信度阈值**：若生成关键实体时模型的平均置信度低于阈值，则触发 fallback（如转人工或返回"不确定"）。

### 4. Chain-of-Verification（CoVe）：自我纠错

让模型分两步走：
1. 先生成初步答案，并列出答案中需要验证的关键陈述。
2. 对每个陈述独立设计验证问题，重新检索或推理，最终根据验证结果修正答案。

研究表明，CoVe 能显著减少事实性错误，尤其在长文本生成场景中效果明显。

### 5. RLHF 定向优化

在人类反馈强化学习（RLHF）阶段，专门设计惩罚幻觉的奖励函数：
- 对"承认无知"的回答给予正向奖励。
- 对"编造信息"的回答给予严厉惩罚。
- 引入对抗性样本（adversarial examples），主动诱导模型犯错后再纠正。

### 6. 多模型投票 / 仲裁

使用一个轻量级的"裁判模型"（Judge Model）对主模型的输出进行事实核查。裁判模型可以是经过 NLI 微调的小模型，也可以是另一个独立的大模型。架构如下：

```
Query → Generator LLM → Draft Answer
                     → Judge LLM (with retrieved docs) → Accept / Reject / Revise
```

## 五、实战场景

### 医疗诊断辅助

**风险**：模型编造药物剂量、误诊病情可能导致生命危险。

**Fallback 设计**：
- 所有建议必须附带权威文献引用（如 PubMed ID、临床指南章节）。
- 若检索不到匹配的临床证据，系统强制返回"建议咨询专业医师"。
- 关键决策点（如用药、手术方案）需至少两名医生人工确认，AI 仅作为信息聚合工具。

### 法律咨询

**风险**：模型虚构法条、引用已废止的判例可能导致案件败诉。

**Fallback 设计**：
- 构建专属的法律条文知识库，RAG 检索范围限定在现行有效的法律法规内。
- 每条回答末尾自动生成"免责声明 + 法条原文链接"。
- 设置"法条时效性检查"中间件，对引用的法规做生效日期校验。

### 金融研报生成

**风险**：模型捏造财务数据、错误解读财报可能引发投资损失。

**Fallback 设计**：
- 所有数字型数据（营收、利润、估值）必须溯源到原始财报 PDF 的具体页码。
- 引入数值一致性校验器，对比模型提取的数字与表格中的原始值。
- 敏感结论（如"买入"评级）需经过合规团队审核后方可发布。

### 通用高风险场景的 Fallback 原则

| 原则 | 说明 |
|------|------|
| **置信度阈值拦截** | 低于阈值的回答不直接展示给用户，转为"我不确定"或人工接管 |
| **引用强制** | 每个事实陈述必须有可点击的出处链接 |
| **人机协同** | AI 生成草稿，人类审核发布 |
| **审计日志** | 记录每次生成的完整上下文，便于事后追溯 |

## 六、面试话术（30 秒版）

> "LLM 幻觉是指模型生成看似合理但事实错误或无中生有的内容。从验证角度可分为 Intrinsic（与源文档矛盾）和 Extrinsic（无法被源文档验证）两类。
>
> 根源在于训练数据的噪声、自回归生成的概率本质，以及模型缺乏真正的'理解'能力，只是在拟合 token 的条件概率分布。
>
> 检测方面，可以用 NLI 模型做蕴含判断、用知识图谱做事实验证，或用 TruthfulQA/RAGAS 等标准化基准评估。
>
> 缓解方案的核心是'让模型有据可依'：RAG 提供外部事实 grounding，self-consistency 通过多次采样投票过滤随机幻觉，降低温度减少不确定性，Chain-of-Verification 让模型自我纠错。在医疗、法律等高风险场景，必须设计置信度拦截 + 引用强制 + 人机协同的三层 fallback 机制，确保幻觉不会直接传导到最终决策。"
