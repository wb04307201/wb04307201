# 06 性能

> 一句话定位：**把"快"做成可量化、可监控、可回归的工程目标**

## 本模块覆盖

| 主题 | 状态 | 说明 |
|------|------|------|
| Core Web Vitals | ⏳ 待补 | LCP / INP / CLS 三大指标与优化 |
| Lighthouse 评分 | ⏳ 待补 | 评分项拆解 / 自动化检测 |
| 性能监控 | ⏳ 待补 | RUM / APM / Web Vitals 上报 |
| 加载性能 | ⏳ 待补 | 资源优先级 / 预加载 / 关键 CSS |
| 运行时性能 | ⏳ 待补 | 长任务 / 防抖 / Web Worker / OffscreenCanvas |

## 与其他模块的关系

- 上游：[05-architecture](../05-architecture/)
- 下游：与 [07-security](../07-security/) 协同（性能优化常涉及安全开关取舍）

## 学习建议

- 性能优化前先**度量**——没有数据支撑的"优化"常常是负优化
- Core Web Vitals 是行业基线，建议优先攻克 LCP 与 INP
