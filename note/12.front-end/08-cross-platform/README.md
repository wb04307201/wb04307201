# 08 跨端

> 一句话定位：**一份代码 / 一支团队,跑遍 Web / 移动 / 桌面 / 小程序**

## 本模块覆盖

| 主题 | 状态 | 说明 |
|------|------|------|
| 移动端 | ⏳ 待补 | React Native / Flutter / Capacitor / KMP 横向对比 |
| 桌面端 | ⏳ 待补 | Electron / Tauri / Wails |
| 小程序 | ⏳ 待补 | 微信 / 支付宝 / 抖音 / Taro / Uni-app |
| PWA | ⏳ 待补 | Service Worker / Web App Manifest / Push |
| WebView 桥接 | ⏳ 待补 | JSBridge / Schema / Hybrid 框架 |

## 与其他模块的关系

- 上游：[05-architecture](../05-architecture/)
- 下游：与 [09-frontend-and-ai](../09-frontend-and-ai/) 协同（AI 能力常作为跨端差异化卖点）

## 学习建议

- 跨端选型要回到「业务场景」——预算、团队技术栈、目标用户设备分布
- WebView 桥接是大多数 Hybrid 框架的底层,值得花 1-2 天理解
