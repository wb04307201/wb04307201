# 架构图绘制

> 用标准化的方式表达系统架构，帮助团队理解系统结构和组件关系。

## 架构视图模型

- [4+1 视图模型](4+1/README.md) — 逻辑视图、开发视图、进程视图、物理视图 + 场景视图
- [C4 模型](c4-model/README.md) — Context → Container → Component → Code 四层抽象

## 选型建议

| 场景 | 推荐模型 | 理由 |
|------|---------|------|
| 向非技术干系人汇报 | C4 Context | 一张图说清系统边界与外部依赖 |
| 团队内部架构评审 | C4 Container + 4+1 进程视图 | 展示部署拓扑与运行时交互 |
| 详细设计文档 | C4 Component + 4+1 开发视图 | 模块划分与代码组织 |
| 全链路追溯 | 4+1 完整五视图 | 覆盖所有关注点 |
