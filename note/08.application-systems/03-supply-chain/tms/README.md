# TMS · 运输管理系统

> 一份按业务场景梳理的 TMS 速查手册：从"派车靠电话"到"算法调度"的升级指南。

---

## 一、一句话定位

**TMS（Transportation Management System，运输管理系统）**：管理"货从哪里 → 到哪里 → 怎么运 → 多少成本"的完整链路，是物流行业的"调度大脑"。

---

## 二、核心功能（6 大模块）

| 模块 | 功能 | 价值 |
|------|------|------|
| **订单管理** | 订单接收、调度分配 | 自动接单 |
| **车辆调度** | 智能匹配车辆 / 司机 | 减少空驶 15-25% |
| **路径规划** | TSP / VRP 算法优化路线 | 节省油费 10-20% |
| **在途跟踪** | GPS + IoT + 电子围栏 | 实时位置 |
| **签收管理** | 电子签收、POD 回单 | 减少纸质单据 |
| **费用结算** | 自动计费、对账、发票 | 减少财务纠纷 |

---

## 三、TMS 在供应链中的位置

```
下单 ──→ 仓库拣货 ──→ TMS（调度）──→ 配送 ──→ 签收
   │            │             │             │         │
订单系统    WMS         TMS          GPS+司机      POD
```

---

## 四、典型行业应用

| 行业 | 重点 |
|------|------|
| **快递/快运** | 末端配送、面单调度 |
| **城配/冷链** | 温控监控、路线优化 |
| **大宗物流** | 整车调度、回程配货 |
| **跨境物流** | 关务、清关、海外仓 |
| **即时配送** | 众包调度（美团/饿了么）|

## 七、核心算法

| 算法 | 用途 |
|------|------|
| **VRP（车辆路径问题）** | 多车辆多订单最优路径 |
| **TSP（旅行商问题）** | 单车辆多点位最优路径 |
| **Dijkstra / A*** | 最短路径 |
| **遗传算法 / 蚁群** | 大规模调度 |
| **机器学习** | 需求预测 / 智能调度 |

---

## 八、实施关键点

1. **数据基础**：地址库 / 车辆库 / 司机库必须标准化
2. **物联网集成**：GPS / 温控 / 称重设备
3. **异常处理**：延误 / 货损 / 拒收的预案
4. **结算规则**：按重量 / 体积 / 距离 / 时效的复杂计费
5. **AI 增强**：需求预测 / 智能调度 / 异常预警
