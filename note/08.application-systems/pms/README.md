## 一、一句话定位

**PMS（Project Management System，项目管理系统）**：在 PMBOK/PRINCE2 等方法论指导下，对项目从立项、计划、执行、监控到收尾的全生命周期管理，目标是"按时、按质、按预算"交付项目。

---

## 二、核心功能（6 大模块）

| 模块 | 功能 | 价值 |
|------|------|------|
| **项目立项** | 项目申请、审批、章程 | 规范启动 |
| **计划管理** | WBS 分解、甘特图、里程碑 | 路径清晰 |
| **任务管理** | 任务分配、进度跟踪、依赖 | 协同执行 |
| **资源管理** | 人力 / 设备分配 | 资源最优 |
| **风险管理** | 风险登记、应对、跟踪 | 主动预警 |
| **收尾管理** | 验收、复盘、知识沉淀 | 经验复用 |

---

## 三、PMS vs 工单系统 vs OA

| 系统 | 关注点 |
|------|--------|
| **PMS** | 项目级（多任务/多阶段/多资源） |
| **工单系统** | 任务级（单次响应） |
| **OA** | 流程级（行政审批） |

## 八、实施关键点

1. **方法论先行**：先选定方法论（瀑布/敏捷），再选工具
2. **模板化**：建立项目模板（立项/WBS/风险/复盘）
3. **资源日历**：人力 / 设备资源池统一管理
4. **挣值管理（EVM）**：进度 × 成本的综合衡量
5. **复盘机制**：项目结束后必须做复盘（5 Why / Lessons Learned）
