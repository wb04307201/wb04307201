# OMS · 订单管理系统

> 一份按业务场景梳理的 OMS 速查手册：从"多渠道接单"到"全渠道订单中枢"。

---

## 一、一句话定位

**OMS（Order Management System，订单管理系统）**：在全渠道零售场景下，把天猫、京东、抖音、小程序、自营 App 等所有渠道的订单统一接入、合并、路由到合适的仓库发货的系统。

---

## 二、为什么需要 OMS（电商场景）

```
订单来源：
  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
  │ 天猫 │ │ 京东 │ │ 抖音 │ │小程序│ │ 自营 │
  └─┬────┘ └─┬────┘ └─┬────┘ └─┬────┘ └─┬────┘
    │         │         │         │         │
    └─────────┴─────────┴─────────┴─────────┘
                       ↓
                    ┌─────┐
                    │ OMS │  ← 统一订单中枢
                    └──┬──┘
                       ↓
       ┌───────────────┼───────────────┐
       ↓               ↓               ↓
   ┌──────┐       ┌──────┐       ┌──────┐
   │ 仓库A│       │ 仓库B│       │ 门店  │
   └──────┘       └──────┘       └──────┘
```

**没有 OMS**：每个渠道订单独立处理，库存不同步，超卖严重。
**有了 OMS**：订单统一管理，库存自动路由（最近仓库发货），降低物流成本。

---

## 三、核心功能（5 大模块）

| 模块 | 功能 | 价值 |
|------|------|------|
| **多渠道接单** | API/ERP 集成 30+ 平台 | 统一订单视图 |
| **订单合并/拆单** | 同用户多订单合并 | 降低运费 15-30% |
| **智能路由** | 最近仓库 / 库存最多 / 最便宜 | 物流时效 +20% |
| **库存同步** | 实时库存共享 | 减少超卖 80% |
| **异常处理** | 缺货 / 退货 / 拒收 | 自动化处理 |

## 四、典型行业应用

| 行业 | 重点 |
|------|------|
| **电商** | 全渠道库存同步、跨平台发货 |
| **新零售** | 线上线下融合（OMO）、门店发货 |
| **跨境电商** | 多仓库路由（国内/海外） |
| **B2B** | 大宗订单、批次管理 |

## 七、核心能力：智能路由

| 策略 | 适用场景 |
|------|---------|
| **最近仓库** | 物流时效优先（次日达）|
| **库存最多** | 避免缺货 |
| **最便宜** | 成本优先（运费最低）|
| **最快发货** | 紧急订单 |
| **定制规则** | 品牌方指定仓库 |

---

## 八、实施关键点

1. **库存主数据**：所有渠道实时同步（避免超卖）
2. **订单合并规则**：用户/收货地址/支付方式等
3. **异常处理**：缺货时自动转单 / 部分发货
4. **对账规则**：与各渠道自动对账（财务结算）
5. **大促准备**：双 11/618 前必须压测（订单洪峰）
