# SCADA · 数据采集与监控系统

> 一份按业务场景梳理的 SCADA 速查手册：工业自动化的"眼睛 + 神经"。

---
## 引言：系统选型

SCADA · 数据采集与监控系统 是企业里绕不开的核心系统之一。

本篇用'它是什么 / 解决什么 / 主流厂商对比'三段式切入，给采购/选型/对接一个起点。

---

## 一、一句话定位

**SCADA（Supervisory Control and Data Acquisition，数据采集与监控系统）**：工业自动化的"中枢神经"—— 实时采集设备数据、监控运行状态、提供远程控制，是 MES/MOM 系统的"数据源"。

---

## 二、核心功能（4 大模块）

| 模块 | 功能 | 价值 |
|------|------|------|
| **数据采集（DAQ）** | 从 PLC/传感器采集实时数据 | 毫秒级采集 |
| **数据监控** | 实时画面显示、报警、趋势 | 异常秒级响应 |
| **远程控制（SCADA）** | 远程启停设备、参数调整 | 减少人工巡检 |
| **历史数据（HMI）** | 数据归档、查询、报表 | 事后追溯 + 分析 |

---

## 三、SCADA vs DCS vs PLC 三层架构

```
┌─────────────────────────────────────────┐
│       管理层（ERP / MES / MOM）            │  ← 决策层
└─────────────────┬───────────────────────┘
                  │
┌─────────────────┴───────────────────────┐
│       监控层（SCADA / HMI）                │  ← SCADA 在这层
└─────────────────┬───────────────────────┘
                  │
┌─────────────────┴───────────────────────┐
│       控制层（DCS / PLC / 传感器）         │  ← 设备控制
└─────────────────────────────────────────┘
```

| 层 | 系统 | 角色 |
|----|------|------|
| 管理层 | ERP/MES/MOM | 业务决策 |
| **监控层** | **SCADA** | **数据采集、监控、报警** |
| 控制层 | DCS / PLC | 设备实时控制 |

## 五、核心技术

| 技术 | 说明 |
|------|------|
| **OPC UA** | 工业设备互联标准协议 |
| **Modbus** | 经典工业通信协议（串口/以太网） |
| **PLC** | 可编程逻辑控制器（数据源） |
| **RTU** | 远程终端单元（野外设备） |
| **HMI** | 人机界面（监控画面）|
| **Historian** | 工业历史数据库（时序数据） |

## 七、选型建议

| 场景 | 推荐 |
|------|------|
| 大型工厂（千点以上）| Siemens / Rockwell |
| 中型工厂（百-千点）| 国内主流（力控 / 亚控）|
| 小型项目（百点以下）| 开源 OpenSCADA / 紫金桥免费版 |
| 云端监控 | AWS IoT / Azure IIoT |
| 跨平台数据采集 | OPC UA 网关 + 主流 SCADA |

---

## 八、实施关键点

1. **网络规划**：工业网络要分层（控制层 / 监控层 / 管理层），做好隔离
2. **数据采集频率**：根据工艺要求选择毫秒级 / 秒级 / 分钟级
3. **报警策略**：分级（紧急 / 重要 / 一般）+ 防抖 + 短信/微信推送
4. **历史数据**：用专门时序数据库（InfluxDB / TimescaleDB）
5. **网络安全**：SCADA 容易被攻击（如 Stuxnet 震网病毒），要严格隔离 + 加密
