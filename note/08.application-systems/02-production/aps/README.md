# APS · 高级计划与排程系统

> 一份按业务场景梳理的 APS 速查手册：从"经验排产"到"算法排产"的升级指南。

---

## 一、一句话定位

**APS（Advanced Planning and Scheduling，高级计划与排程系统）**：用数学算法（约束规划 / 遗传算法 / 强化学习）解决"在有限产能下，如何让生产计划最优"的系统，是 ERP/MES 的大脑。

---

## 二、核心价值（为什么需要 APS）

| 传统排产 | APS 排产 |
|---------|---------|
| 靠老师傅经验 | 算法自动求解 |
| 大批量批次、库存高 | 小批量精准、库存低 |
| 换线时间长 | 智能排产减少换线 |
| 紧急插单靠吼 | 系统自动重排 |
| 计划变更 1 周 | 计划变更 1 小时 |

**典型收益**：
- 库存降低 20-30%
- 交付准时率提升 15-25%
- 产能利用率提升 10-20%
- 紧急插单响应从 4 小时 → 30 分钟

---

## 三、核心功能（5 大模块）

| 模块 | 功能 | 算法 |
|------|------|------|
| **主生产计划（MPS）** | 月/周生产计划 | 滚动计划 |
| **物料需求计划（MRP）** | 计算物料需求 | BOM 展开 |
| **车间排程（APS）** | 分钟级工序排程 | 约束规划 / 遗传算法 |
| **订单承诺（ATP/CTP）** | 订单可承诺交期 | 产能平衡 |
| **模拟与 What-If** | 模拟不同方案 | 数字孪生 |

---

## 四、核心算法

| 算法 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **约束规划（CP）** | 中小规模排程 | 求解快、可解释 | 大规模慢 |
| **遗传算法（GA）** | 大规模排程 | 全局最优 | 求解慢、参数敏感 |
| **模拟退火** | 复杂约束 | 跳出局部最优 | 收敛慢 |
| **强化学习** | 动态环境 | 自适应 | 需要大量数据 |
| **数字孪生** | 复杂工厂 | 全局可视化 | 实施成本高 |

---

## 五、与 ERP/MES 的关系

```
                    ┌──────────┐
                    │   APS    │  ← 大脑（做什么 / 何时做 / 用什么资源）
                    └────┬─────┘
                         │
       ┌─────────────────┼─────────────────┐
       ↓                 ↓                 ↓
    ┌──────┐         ┌──────┐          ┌──────┐
    │ ERP  │         │ MES  │          │ WMS  │
    │资源/成本│         │工单执行│          │物料配送│
    └──────┘         └──────┘          └──────┘
```

- **ERP** 提供订单、库存、成本
- **APS** 在 ERP 数据基础上生成最优计划
- **MES** 执行 APS 的车间排程

## 七、选型建议

| 场景 | 推荐 |
|------|------|
| 大型集团 | SAP IBP / Blue Yonder |
| 中型制造（年营收 1-10 亿）| Dassault DELMIA / 鼎捷 |

| 离散制造（多品种小批量）| 遗传算法 APS |
| 流程制造（大宗）| 约束规划 APS |

## 八、实施关键点

1. **数据基础**：必须有准确的 BOM、工艺路线、设备产能
2. **场景范围**：先做"瓶颈工位排程"再扩展
3. **算法选择**：从约束规划开始（简单可解释）
4. **人机协同**：算法是参考，最终决策仍由计划员做
5. **持续优化**：积累数据后逐步引入强化学习

---

## 九、APS 与上下游系统深度集成

APS 作为工厂的"大脑"，其排程质量直接依赖数据输入的准确性——与 MES、MOM、ERP 的集成深度决定了 APS 的实际效果。

### 9.1 集成数据流

```
ERP（订单 + 物料 + 成本）
  ↓ 提供需求与约束
APS（最优排程计算）
  ↓ 下发排程计划
MOM（运营管理 + 调度）
  ↓ 分解为工单
MES（车间执行 + 实时反馈）
  ↑ 回传实际进度 → APS 重排
```

### 9.2 APS 与 MES/MOM/ERP 的边界

| 系统 | 计划层级 | 时间粒度 | 核心问题 |
|------|---------|---------|---------|
| **ERP** | 月/季度计划 | 月 | "做什么、做多少" |
| **APS** | 日/班次排程 | 小时/分钟 | "什么时候做、用哪台设备" |
| **MOM** | 车间调度 | 分钟 | "怎么分配资源、怎么协调" |
| **MES** | 工单执行 | 秒/实时 | "现在做到哪了、有没有异常" |

### 9.3 APS 上线路径建议

| 阶段 | 内容 | 周期 |
|------|------|------|
| **Phase 1** | 先上 MRP（物料需求）+ 简单排程 | 3-6 月 |
| **Phase 2** | 上约束规划排程（瓶颈工位） | 6-12 月 |
| **Phase 3** | 引入遗传算法 / 强化学习 | 12-18 月 |
| **Phase 4** | 数字孪生 + 全自动排程 | 18-24 月 |
