# PMS · 项目管理系统

> 一份按业务场景梳理的 PMS 速查手册：从"Excel 管项目"到"专业项目管理平台"。

---

## 一、一句话定位

**PMS（Project Management System，项目管理系统）**：在 PMBOK/PRINCE2 等方法论指导下，对项目从立项、计划、执行、监控到收尾的全生命周期管理，目标是"按时、按质、按预算"交付项目。

---

## 二、核心功能（6 大模块）

| 模块 | 功能 | 价值 |
|------|------|------|
| **项目立项** | 项目申请、审批、章程 | 规范启动 |
| **计划管理** | WBS 分解、甘特图、里程碑 | 路径清晰 |
| **任务管理** | 任务分配、进度跟踪、依赖 | 协同执行 |
| **资源管理** | 人力 / 设备分配 | 资源最优 |
| **风险管理** | 风险登记、应对、跟踪 | 主动预警 |
| **收尾管理** | 验收、复盘、知识沉淀 | 经验复用 |

---

## 三、PMS vs 工单系统 vs OA

| 系统 | 关注点 |
|------|--------|
| **PMS** | 项目级（多任务/多阶段/多资源） |
| **工单系统** | 任务级（单次响应） |
| **OA** | 流程级（行政审批） |

---

## 四、典型行业应用

| 行业 | 重点 |
|------|------|
| **软件研发** | 敏捷 / Scrum / Kanban |
| **建筑工程** | PMBOK / 进度 / 成本管理 |
| **咨询/服务** | 项目工时 / 人天计费 |
| **研发/产品** | 需求 / 任务 / 缺陷 |
| **营销活动** | 活动排期 / 物料 / 预算 |

---

## 五、核心方法论

| 方法论 | 适用场景 |
|--------|---------|
| **PMBOK（瀑布）** | 传统工程项目 / 大型项目 |
| **PRINCE2** | 政府 / 欧洲项目 |
| **Agile / Scrum** | 软件研发 / 迭代型 |
| **Kanban（看板）** | 运维 / 持续交付 |
| **SAFe** | 大规模敏捷（跨团队）|
| **PMBOK + Agile 混合** | 渐进式转型 |

---

---

## 八、实施关键点

1. **方法论先行**：先选定方法论（瀑布/敏捷），再选工具
2. **模板化**：建立项目模板（立项/WBS/风险/复盘）
3. **资源日历**：人力 / 设备资源池统一管理
4. **挣值管理（EVM）**：进度 × 成本的综合衡量
5. **复盘机制**：项目结束后必须做复盘（5 Why / Lessons Learned）

---

## 九、与研发岗位的关系

研发工程师最常用的 PMS：
- **Jira**（敏捷 / Scrum）
- **飞书项目 / 钉钉项目**（国内协作）
- **GitHub Projects / GitLab Issues**（与代码集成）
