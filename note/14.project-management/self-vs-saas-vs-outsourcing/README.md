# 技术选型 ROI：自研 vs SaaS vs 外包 决策树

> CTO / 技术总监最常拍板的一类决策：花 50 万买 SaaS / 花 30 万外包 / 花 200 万自研，到底哪个 ROI 最高？考察的是 **全生命周期 TCO 计算 + 决策树判断 + AI 时代的额外变量**。

> **前置知识**：[app-quote-breakdown](../app-quote-breakdown/README.md)（项目成本拆解）、[outsourcing-pitfalls](../outsourcing-pitfalls/README.md)（外包隐性成本）、[mobile-tech-stack](../../09.front-end/08-cross-platform/mobile-tech-stack/README.md)（前端跨端选型）。

---

## 引言：SaaS 推销员 vs 老板 vs CTO

```text
周一：
  SaaS 推销员："用我们的产品，5 万一年免运维，不用自建团队！"
  外包团队："30 万做一次，包含 1 年维护。"
  老板："我们想自己造，招聘 3 个工程师，200 万一年。"

周五：
  老板迷茫："拍板三选一，应该怎么看？"

老陈（CTO）3 步走：
  1. 算 TCO（5 年总拥有成本）
  2. 算 ROI（业务价值 / 总投入）
  3. 走决策树（5 大维度：能力 / 时间 / 定制 / 数据 / 规模）
```

**真相**：技术选型不是"便宜 vs 贵"，是 **"你的总投入 vs 你能用多久 / 撑起多少业务"**。5 万的 SaaS 用 1 年 = 5 万；200 万的自研撑 3 年 = 66 万/年；30 万的外包只能撑 1 年 = 30 万/年。

**"看似便宜"可能最贵** —— 不算全生命周期 = 算错账。

---

## 一、核心结论（TL;DR）

| 维度 | 自研 | SaaS | 外包 |
|------|------|------|------|
| **首年成本** | 200-500 万（招聘 + 启动） | 5-50 万/年 | 30-100 万 |
| **次年起** | 50-100 万/年（人力维护） | 5-50 万/年 | 30 万（维护）/ 重做 |
| **上线时间** | 3-12 个月 | 1-2 周 | 1-3 个月 |
| **定制化** | ⭐⭐⭐⭐⭐ 完美匹配 | ⭐⭐ 受限于产品 | ⭐⭐⭐⭐ 可改 |
| **团队能力沉淀** | ⭐⭐⭐⭐⭐ 长期持有 | ⭐ 无 | ⭐⭐ 项目结束就散 |
| **数据/合规** | ⭐⭐⭐⭐⭐ 完全可控 | ⭐⭐⭐ 受 SaaS 厂商限制 | ⭐⭐⭐ 视外包商能力 |
| **运营负担** | ⭐⭐⭐⭐⭐ 团队全职 | ⭐ 几乎无 | ⭐⭐ 沟通成本 |
| **风险** | 招聘失败 / 留不住人 | 厂商倒闭 / 涨价 | 项目跑路 / 烂尾 |
| **最适合** | 核心业务 + 长期投入 | 标准流程 + 通用场景 | 短期项目 + 临时需求 |

> **一句话**：**SaaS 优先 → 外包补位 → 自研核心**。能用 SaaS 解决的，绝对不自己造；只有 SaaS 撑不住的、才考虑外包或自研。

---

## 二、5 大评估维度

每类选型都过这 5 个维度，**每个维度按 0-5 打分**，总分 ≥ 20 才算"上选"，否则放弃。

### 维度 1：业务匹配度（核心权重 30%）

| 分值 | 标准 |
|------|------|
| 0-1 | SaaS 完全满足（10 个里满足 9.5 个）|
| 2-3 | SaaS 大体满足，需定制 |
| 4-5 | SaaS 撑不住，外包 / 自研 |

> **关键问题**：把需求列出来，**逐项对照 SaaS demo**。90% 需求 SaaS 已覆盖的 → 别造轮子。

### 维度 2：团队能力（核心权重 25%）

| 分值 | 标准 |
|------|------|
| 0-1 | 当前团队不熟悉，无能力积累 |
| 2-3 | 团队有 1-2 名该领域老兵 |
| 4-5 | 团队已是该领域专家 |

> **真相**：自研 = 必须有"能 hold 住"的人。**没人 = 别自研**。
> **AI 时代修正**：Cursor + Claude Code 这类 AI Coding 工具，让 2 人能 hold 住原本 5 人团队的工作；但 Prompt/Context Engineering 这套新能力也得有人会。

### 维度 3：上线时间（核心权重 20%）

| 分值 | 标准 |
|------|------|
| 0-1 | 3 月内能上线 |
| 2-3 | 3-6 月 |
| 4-5 | 6+ 月，且等不及 |

> **关键问题**：业务 **等不等得起**？如果错过这 2 周窗口，市场被对手占完 —— 用 SaaS。
> 如果是个内部工具，慢慢做也行 —— 可以外包。

### 维度 4：数据/合规（核心权重 15%）

| 分值 | 标准 |
|------|------|
| 0-1 | 公开数据 / 非敏感 |
| 2-3 | 一般业务数据 |
| 4-5 | **核心机密 / 强合规**（金融、医疗、政企）|

> **真相**：涉及核心数据 / GDPR / HIPAA / 数据出境 —— 必须私有化部署或自研，SaaS 不能碰。

### 维度 5：长期 ROI（核心权重 10%）

| 分值 | 标准 |
|------|------|
| 0-1 | 用 1 年就换 |
| 2-3 | 用 3-5 年 |
| 4-5 | 用 5 年+，是公司核心 |

> **关键问题**：用 1 年的项目，**没必要自研**（5 年回本不可能）；用 5 年+ 的核心，才值得自研。

## 三、决策树（5 步走）

```text
第一步：业务匹配度高（≥ 4）？
  ├─ 是 → 走第二步
  └─ 否 → 直接选 SaaS（退出）

第二步：数据合规要求严（≥ 4）？
  ├─ 是 → 走第三步（必须自研或私有化）
  └─ 否 → 走第四步

第三步：团队有 hold 住的能力（≥ 3）？
  ├─ 是 → 走第五步（选 SaaS 私有化部署 OR 自研）
  └─ 否 → 外包给有合规经验的服务商（候选 A）+ 招 1 个自研负责人（candidate B，2-3 年过渡到自研）

第四步：上线时间紧（≤ 2 月）？
  ├─ 是 → 选 SaaS
  └─ 否 → 走第五步

第五步：长期 ROI 高（≥ 4，用 5 年+）？
  ├─ 是 → 自研
  └─ 否 → 外包
```

### 实战场景对照表

| 业务场景 | 推荐选型 | 决策依据 |
|---------|---------|---------|
| 公司内部 IM | **SaaS**（钉钉/企业微信/飞书） | 业务匹配 5、时间紧、ROI 中 |
| 营销官网 | **SaaS**（WordPress/Notion/Squarespace） | 业务匹配 5、SaaS 已覆盖 |
| 行业垂直 CRM | **外包**（中小型项目） | 业务匹配 4、SaaS 撑不住、外包可控 |
| 公司核心订单系统 | **自研** | 业务匹配 5、合规严、长期 ROI 5 |
| 数据中台 | **自研 + 开源** | 业务匹配 5、合规严、长期 ROI 5 |
| 一次性的活动报名 | **外包** | 业务匹配 4、用 1 次、ROI 低 |
| 内部数据看板 | **SaaS**（Metabase/Tableau）| 业务匹配 5、SaaS 撑住 |
| 跨公司协作平台 | **自研 + 外包** | 业务匹配 5、合规 5、外包搭基线、自研做核心 |

## 四、TCO 计算公式（5 年视角）

### 总拥有成本公式

```text
TCO（5 年）= 启动成本 + Σ(年运营成本 × 5 年)
```

### 3 类方案的 TCO 拆解

#### 自研

```text
启动：招聘 3 人 × 6 个月 onboarding + 20 万服务器+licenses
       ≈ 80 万
年运营：3 人 × 50 万薪资 + 30 万服务器+licenses
       ≈ 180 万/年
5 年 TCO ≈ 80 + 180 × 5 = 980 万
```

#### SaaS

```text
启动：0（注册即用）
年运营：10 万/年（订阅）+ 3 万集成费 = 13 万/年
5 年 TCO ≈ 13 × 5 = 65 万
```

#### 外包

```text
启动：30 万开发 + 5 万需求沟通
年运营：8 万/年维护 + 5 万/年沟通 = 13 万/年
5 年 TCO ≈ 35 + 13 × 5 = 100 万
```

### ROI 公式

```text
ROI = (业务价值 - TCO) / TCO × 100%
```

例如订单系统：
- 业务价值：每单省 0.5 元 × 1000 万单/年 = 500 万/年
- 5 年价值 = 2500 万
- 自研 TCO = 980 万 → ROI = (2500 - 980) / 980 × 100% = **155%** ✅
- SaaS TCO = 65 万 → 但实现不了核心需求 → **无效**
- 外包 TCO = 100 万 → 但撑不住 5 年演进 → 第 3 年重做 → 实际 TCO ≈ 250 万 → ROI ≈ **900%** 看似高实则难续

## 五、AI 时代的 3 个额外变量（2026+）

### 变量 1：AI Coding 加速自研周期

```text
过去：自研 12 个月
现在：Cursor + Claude Code + 2 个资深 → 自研 4-6 个月
```

→ **过去"自研 12 个月"的判断被 AI Coding 颠覆了**。当自研周期缩短到 4-6 个月，与外包差异缩小，纯成本对比要重新算。

### 变量 2：AI 训练/推理的额外成本

- 自研若用 LLM：**推理成本 × 调用次数**（参考 [12.story/36a](../../12.story/36a-ai-token-cost-structure.md)）
- SaaS 通常含 LLM：成本打包进订阅费
- 外包不能碰 LLM：**经常需要二次开发**

### 变量 3：AI Coding 的人才账本

- 团队是否 **会用 Harness + Cursor + Agent**？
- 如果不会，"团队能力"维度要降 1-2 分
- 如果会，但 SaaS 定制能力有限 → 自研 ROI 上升

## 六、实战 checklist（决策前 5 项）

1. **需求清单 + 5 年规划**：把"今天 + 3 年后"的需求都列出来
2. **SaaS 全市场调研**：选 3 家厂商对照清单，**不能只看一家**
3. **外包交付能力评估**：看合同 → 合同模板见 [outsourcing-pitfalls](../outsourcing-pitfalls/README.md)
4. **自研能力盘点**：列出团队"已经会 / 可以学 / 学不会"的能力
5. **AI 时代修正**：上面 4 项的每个数字，**重新过一遍"AI Coding 加速 / 推理成本"** 这两个变量

## 七、避坑指南

| 坑 | 后果 | 避坑 |
|----|------|------|
| ① **盲目自研**：老板说"有数据就得自研" | 招不到人 / 招到了留不住 / 项目烂尾 | 走决策树；如果自研是 4-5 分而非 5 分，先 SaaS |
| ② **盲目 SaaS**：CTO 说"用现成的就行" | 需求撑不住 / 关键数据不可控 | 5 大维度逐一过；合规问题不是"以后再说" |
| ③ **盲目外包**：项目紧就找人做 | 维护没人 / 二次开发起价翻倍 | 必须看清合同 8 条 + 维护 SLA |
| ④ **"AI Coding 万能论"** | 啥都能 AI 写，自研成本估算减半 | AI 加速的是"写"；**需求/设计/测试" 不会自动 AI** |
| ⑤ **"一旦造了不能换"** | 选型=赌博 | 第一版用 SaaS，跑 1 年后看真实需求；**避免自研 lock-in** |
