## 核心概念

- **定义** — 算法运行时临时占用的额外存储空间（**不包括输入数据本身**）与输入规模 `n` 的函数关系
- **关键规则** — 忽略常数、保留最高阶、输入空间不计

## 常见复杂度分类

| 复杂度 | 示例 | 说明 |
|--------|------|------|
| **O(1)** | 迭代计算、交换变量 | 额外空间恒定 |
| **O(log n)** | 分治算法递归栈 | 递归深度对数级 |
| **O(n)** | 动态数组、线性递归 | 与输入规模成正比 |
| **O(n²)** | 二维矩阵、嵌套递归 | 平方增长 |

## 影响因素

| 因素 | 说明 | 示例 |
|------|------|------|
| **数据结构** | 不同结构的空间开销差异 | 哈希表需额外空间处理冲突 |
| **递归深度** | 递归算法的空间由调用栈深度决定 | 二分查找递归版 O(log n) |
| **辅助空间** | 算法运行中申请的临时空间 | 归并排序 O(n) 临时数组 |

## 排序算法空间对比

| 算法 | 空间复杂度 | 说明 |
|------|-----------|------|
| 冒泡排序 | O(1) | 原地排序 |
| 快速排序 | O(log n) | 递归栈（平均） |
| 归并排序 | O(n) | 需临时数组 |

## 优化策略

| 策略 | 方法 | 效果 |
|------|------|------|
| **原地算法** | 修改输入而非创建副本 | 反转数组：双指针 O(1) |
| **迭代替代递归** | 手动管理栈消除递归 | 斐波那契：O(n) → O(1) |
| **流式处理** | 分块处理大数据 | 大文件排序：分块→归并 |

## 代码示例

**递归阶乘 — 空间 O(n)**
```python
def factorial_recursive(n):
    if n == 0:
        return 1
    return n * factorial_recursive(n - 1)  # 栈深度为 n
```

**迭代阶乘 — 空间 O(1)**
```python
def factorial_iterative(n):
    result = 1
    for i in range(1, n + 1):
        result *= i  # 仅需固定数量变量
    return result
```

## 注意事项

- **输入空间不计** — 只分析额外申请的空间
- **递归开销** — 树形递归可能达 O(2ⁿ)
- **实际内存** — 复杂度是理论上限，受数据类型/系统策略影响
