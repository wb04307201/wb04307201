# 时间复杂度

> 时间复杂度衡量算法执行时间随输入规模增长的变化速率，用大 O 符号表示，是算法设计和优化的核心依据。

---

## 核心概念

- **定义** — 算法在最坏/平均/最好情况下，基本操作次数与输入规模 `n` 的函数关系
- **目的** — 忽略常数因子和低阶项，聚焦效率的**增长趋势**

## 常见复杂度分类

| 复杂度 | 示例 | 说明 |
|--------|------|------|
| **O(1)** | 访问数组元素 | 执行时间恒定，与输入无关 |
| **O(log n)** | 二分查找、堆操作 | 每次减半，对数增长 |
| **O(n)** | 线性遍历 | 与输入规模成正比 |
| **O(n log n)** | 快速排序、归并排序 | 分治算法典型复杂度 |
| **O(n²)** | 冒泡排序、双重循环 | 嵌套循环，平方增长 |
| **O(2ⁿ)** | 递归斐波那契 | 指数增长，需谨慎 |
| **O(n!)** | 旅行商问题（暴力） | 阶乘增长，仅限极小规模 |

## 关键规则

1. **忽略常数** — `O(2n)` → `O(n)`
2. **保留最高阶** — `O(n² + n)` → `O(n²)`
3. **多变量** — `O(mn)` 需明确变量关系

## 优化策略

| 策略 | 方法 | 效果 |
|------|------|------|
| 降低复杂度 | O(n²) 排序 → O(n log n) | 大规模数据显著加速 |
| 空间换时间 | 哈希表替代线性查找 | O(n) → O(1) |
| 预处理 | 构建索引/缓存 | 查询阶段大幅加速 |

## 代码示例

**线性搜索 — O(n)**
```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1
```

**二分查找 — O(log n)**
```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

## 注意事项

- 通常分析**最坏情况**（如快速排序最坏 O(n²)）
- 平均情况需结合概率分布
- 复杂度是理论上限，实际性能受硬件/数据分布影响
