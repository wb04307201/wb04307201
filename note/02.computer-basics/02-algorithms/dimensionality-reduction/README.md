# 降维算法

> 在保留关键信息的同时减少特征维度，缓解维度灾难、降低计算成本、便于可视化。

## 为什么需要降维？

- **维度灾难**：特征维度越高，数据点间距离趋于均匀，模型难以区分
- **计算成本**：高维数据的存储和计算开销指数级增长
- **可视化需求**：人类只能理解 2D/3D，降维后才能直观观察数据分布
- **噪声过滤**：低方差维度往往携带噪声，降维可自动过滤

## 5 大降维算法对比

| 算法 | 类型 | 核心思想 | 保留结构 | 可解释性 | 适用场景 |
|------|------|---------|---------|---------|---------|
| **PCA** | 线性 | 正交投影到方差最大方向 | 全局方差 | ✅ 高（主成分载荷） | 数据压缩、去相关、预处理 |
| **LDA** | 线性 | 最大化类间距离 / 最小化类内距离 | 类别可分性 | ✅ 高 | 分类预处理、人脸识别 |
| **t-SNE** | 非线性 | 高维相似度 → 低维 t 分布保持 | 局部邻域 | ❌ 低（随机初始化） | 高维数据可视化（2D/3D） |
| **UMAP** | 非线性 | 拓扑流形学习 + 模糊集合 | 局部 + 全局 | ❌ 低 | 可视化（比 t-SNE 快、保留全局结构） |
| **Autoencoder** | 非线性（神经网络） | 编码器压缩 → 解码器重建 | 重建误差最小 | ❌ 低（黑盒） | 特征学习、生成模型 |

## 选型决策树

```text
目标是什么？
├── 可视化（2D/3D）
│   ├── 需保留全局结构 → UMAP
│   └── 只看局部簇 → t-SNE
├── 分类预处理 → LDA（有标签）/ PCA（无标签）
├── 数据压缩 / 去相关 → PCA
└── 特征学习（深度学习） → Autoencoder / VAE
```

## 关键概念

- ** explained variance（解释方差）**：PCA 中每个主成分捕获的方差占比，前 k 个主成分的累计解释方差决定降维后信息保留程度
- **困惑度（Perplexity）**：t-SNE 核心参数，近似"有效邻居数"，通常设 5-50，过大/过小都会扭曲结构
- **重建误差**：Autoencoder 的损失函数，衡量压缩后信息丢失程度
