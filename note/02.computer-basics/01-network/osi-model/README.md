# OSI 七层模型

> OSI（Open Systems Interconnection）是由 ISO 提出的**理论参考模型**，采用七层结构，从物理传输到应用服务逐层抽象，每层各司其职、依赖下层、服务上层。

---

## 七层结构总览

```
┌─────────────────────────────────────────────┐
│ 7. 应用层    Application    HTTP/DNS/SMTP    │  ← 面向用户
│ 6. 表示层    Presentation   SSL/TLS/JPEG     │  ← 数据格式/加密
│ 5. 会话层    Session        NetBIOS/RPC      │  ← 会话管理
│ 4. 传输层    Transport      TCP/UDP          │  ← 端到端可靠传输
│ 3. 网络层    Network        IP/ICMP/OSPF     │  ← 路由寻址
│ 2. 数据链路层 Data Link     Ethernet/VLAN    │  ← 帧传输/MAC寻址
│ 1. 物理层    Physical       RJ45/光纤/WiFi   │  ← 比特流传输
└─────────────────────────────────────────────────┘
```

---

## 各层详解

### 1. 物理层（Physical Layer）
- **功能** — 定义机械/电子/电气特性，传输比特流（0/1）
- **协议** — 以太网（RJ-45）、WiFi、蓝牙、USB、RS-232
- **设备** — 集线器、中继器

### 2. 数据链路层（Data Link Layer）
- **功能** — 比特流封装为**帧（Frame）**，实现节点间可靠传输
  - **MAC 子层** — 介质访问控制（CSMA/CD）
  - **LLC 子层** — 帧同步、错误恢复
- **关键机制** — CRC 校验、MAC 地址寻址、流量控制
- **协议** — PPP、HDLC、VLAN、STP
- **设备** — 交换机、网桥

### 3. 网络层（Network Layer）
- **功能** — 跨网络路由与转发，**逻辑地址（IP）** 寻址
- **核心能力** — 路径选择、拥塞控制、分片与重组
- **协议** — IP（IPv4/IPv6）、ICMP（Ping）、OSPF、BGP、NAT
- **设备** — 路由器、三层交换机

### 4. 传输层（Transport Layer）
- **功能** — **端到端（进程间）** 传输，端口号区分应用
- **TCP** — 面向连接、可靠传输（确认/重传/排序/流量控制）
- **UDP** — 无连接、低延迟（视频/游戏/DNS）
- **关键概念** — 端口号（80/443）、套接字（IP+端口）

### 5. 会话层（Session Layer）
- **功能** — 建立/维护/终止应用间会话
- **能力** — 对话控制（全双工/半双工）、同步点恢复
- **协议** — NetBIOS、RPC、PPTP

### 6. 表示层（Presentation Layer）
- **功能** — 数据格式转换、加密/解密、压缩/解压缩
- **协议** — SSL/TLS、JPEG/PNG、ASCII/Unicode、MIME

### 7. 应用层（Application Layer）
- **功能** — 直接面向用户，提供网络服务接口
- **协议** — HTTP/HTTPS、FTP、SMTP/POP3、DNS、DHCP

---

## OSI vs TCP/IP 对照

| OSI 七层 | TCP/IP 四层 | 说明 |
|----------|------------|------|
| 应用层 + 表示层 + 会话层 | **应用层** | 上三层合并 |
| 传输层 | **传输层** | 功能一致 |
| 网络层 | **网络层**（网际层） | 功能一致 |
| 数据链路层 + 物理层 | **链路层**（网络接口层） | 下两层合并 |

---

## 实践价值

1. **标准化** — 不同厂商设备互操作的基础框架
2. **故障排查** — 逐层诊断（物理层断网 → 应用层页面打不开）
3. **协议设计** — 指导新协议开发，功能分层明确

## 案例：访问网页的数据旅程

```
浏览器输入 URL
  → 应用层: HTTP 请求
  → 表示层: TLS 加密
  → 会话层: 建立会话
  → 传输层: TCP 分段 + 端口号
  → 网络层: IP 封装 + 路由选择
  → 数据链路层: MAC 帧封装 + CRC
  → 物理层: 比特流通过网线/光纤
```
