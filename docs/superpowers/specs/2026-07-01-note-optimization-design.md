## 4. 子 README 标准模板

### 顶层模块 README（14 份）

```markdown
<!--
module:
  parent: note
  slug: 01.java
  type: index
  category: 后端语言
  summary: <一句话定位本模块>
-->

# <数字编号>、<模块中文名>

> <一句话导览，说明本模块覆盖什么>

> 📘 **写作与维护规范**：[CONTRIBUTING.md](../CONTRIBUTING.md)
> 📚 **返回总目录**：[note 总目录](../README.md)

## 目录导航

| 序号 | 主题 | 核心内容 |
|------|------|----------|
| 1 | [子目录 1](01-sub1/README.md) | 一句话核心 |
| ... | ... | ... |

## 适用人群

（如适用；不是所有模块都需要）

## 学习路径

（如适用；典型 3 段式：入门 / 进阶 / 精通）
```

### 二级子目录 README（几十份）

```markdown
<!--
module:
  parent: <主模块>
  slug: <主模块>/<子目录>
  type: article
  category: <分类>
  summary: <一句话>
-->

# <子目录名>

> <1-2 句本子目录定位>

## 核心内容

| 主题 | 核心问题 |
|------|---------|
| [文章 1](article-1/README.md) | 一句话核心问题 |
| ... | ... |

---

> 📚 **返回主模块**：[主模块名](../README.md)
```

### 三级文章页（按需 frontmatter）

```markdown
<!--
module:
  parent: <主>/<子>
  slug: <主>/<子>/<文章>
  type: article
  category: <分类>
  summary: <一句话>
-->
```

**说明**：
- frontmatter 写在 HTML 注释里，GitHub / Obsidian 双兼容
- 不引入新文件类型（无 .json 索引、无脚本）

## 5. 模块推进顺序

| 批 | 模块 | 选这个的理由 |
|----|------|------------|
| **① 示范批 (3 个)** | `13.split-hairs` / `06.spring` / `01.java` | 三类典型代表：高频 Q&A 卡片型 / 大型结构文档型 / 仓库最重且最有版本压力的超级模块 |
| **② 推广批 (8 个)** | `02 / 03 / 04 / 05 / 09 / 10 / 11 / 12` | 已成熟但风格各异；模板复用率最高 |

**依赖**：批与批之间没有强依赖，可以打乱。**批内模块可以并行**（不同会话/不同 PR 同时进行）。

## 9. 风险与回滚

- **风险**：14 模块 × 56 commit 的工作量极大，单模块改动可能影响其他模块引用
- **回滚策略**：每模块独立 commit，需要时可 `git revert <commit-hash>` 单模块回滚
- **依赖保护**：`note/README.md`（总目录）的章节锚点必须保持稳定；所有改动以"自底向上"为序（先二级子目录、再主模块 README、最后总目录）
