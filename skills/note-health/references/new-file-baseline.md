# 新文件质量基线（new-file-baseline）

本文件是"写作/验收结构模板"——描述一个**新沉淀文件应有的段落结构与写作模式**。它与 `leaf-quality.md` 的"打分维度"（G1-G6 + 模块专属）互补：`leaf-quality.md` 负责**逐维度打分**，本文件负责**结构基线与写作规格**，两者不重复打分。

## 新文件质量基线

**基于沉淀新文件的质量基线**（90%+ 符合）：

```yaml
新沉淀文件质量基线：
  行数：100-200 行（< 80 行 = 浅占位，> 300 行 = 过长）

  必选 7 段结构（所有新文件必备）：
    1. frontmatter（module / question / story 三选一）
    2. H1 标题（一句话定位）
    3. 一句话定位（粗体，30 字内）
    4. 🎯 学习目标（3-5 条）
    5. 📚 章节清单（表格：主题 / 内容 / 时长）
    6. 🔗 兄弟章节（4-6 个互链）
    7. ⚠️ 反直觉（4-5 条）+ ← 返回上级

  可选 3 段（按主题特性选用）：
    8. 📐 核心数学 / 公式（适用：算法 / 物理 / 量化策略 / 性能推导）
    9. 🛠️ 实操代码（适用：工具实践 / 算法实现 / 框架使用，至少 30 行）
    10. 📊 效果对比表（适用：方案选型 / 性能对比，需真实数据）

两种写作模式：
  快改（默认）：
    - 100-200 行
    - 必选 7 段 + 视情况加可选段
    - 30 分钟 / 文件
    - 适合"主模块子文章"
  深耕（按需）：
    - 500-1000 行
    - 多章 + 实操 + 论文复现
    - 2-4 小时 / 文件
    - 适合"经典大专题"（如 12.story 餐厅叙事）

## 适用性说明

**面试题（13.split-hairs）**：通常只需必选 7 段（数学 / 实操 / 对比表非必需）
**算法 / 原理（02.computer-basics / 11.ai）**：8/9/10 视情况（算法常需要数学；框架常需要对比）
**工具实践（05.tools / 07.workflow）**：9 必备（实操命令）
**餐厅叙事（12.story）**：7 段 + 阿明餐厅元素（见 leaf-quality.md D 类规则）
