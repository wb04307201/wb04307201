# 新文件质量基线（new-file-baseline）

本文件是"写作/验收结构模板"——描述一个**新沉淀文件应有的段落结构与写作模式**。它与 `leaf-quality.md` 的"打分维度"（G1-G6 + 模块专属）互补：`leaf-quality.md` 负责**逐维度打分**，本文件负责**结构基线与写作规格**，两者不重复打分。

## 新文件质量基线

**基于沉淀新文件的质量基线**（90%+ 符合）：

```yaml
新沉淀文件质量基线：
  行数：100-200 行（< 80 行 = 浅占位，> 300 行 = 过长）
  必备 10 段结构：
    1. frontmatter（module / question / story 三选一）
    2. H1 标题（一句话定位）
    3. 一句话定位（粗体，30 字内）
    4. 🎯 学习目标（3-5 条）
    5. 📚 章节清单（表格：主题 / 内容 / 时长）
    6. 📐 核心数学 / 公式（如果适用）
    7. 🛠️ 实操代码（Python 至少 30 行）
    8. 📊 效果对比表（真实数据）
    9. 🔗 兄弟章节（4-6 个互链）
    10. ⚠️ 反直觉（4-5 条）+ ← 返回上级

两种写作模式：
  快改（默认）：
    - 100-200 行
    - 标准 10 段模板
    - 30 分钟 / 文件
    - 适合"主模块子文章"
  深耕（按需）：
    - 500-1000 行
    - 多章 + 实操 + 论文复现
    - 2-4 小时 / 文件
    - 适合"经典大专题"（如 12.story 餐厅叙事）
```
